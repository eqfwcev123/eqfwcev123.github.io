브로드캐스트 주소 계산법

브로드캐스팅 주소 계산법 알아야할 용어들: 네트워크 주소: 하나의 네트워크를 통칭하기 위해 사용하는 주소. 브로드캐스 주소: 특정 네트워크에 속하는 모든 호스트/클라이언트 들이 듣게되는 주소를 의미한다. 네트워크 주소와는 반대로 특정 네트워크의 맨 마지막 주소를 브로드캐스르 주소로 사용한다. IP주소는 네트워크 주소와 호스트주소로 나누어진다. IP주소를 네트워크 주소와 호스트 주소로 구분해 주는것이 서브넷 마스크(Subnet mask)이다. 추가적으로 서브넷은 추가적인 서브넷 서브 네트워크가 필요할 경우 IP주소 중 호스트 주소를 서브넷과 호스트주소(<네트워크주소><서브넷><호스트주소>) 로 나누어준다. 서브넷 마스크라고 불리는 이유는 서브넷 마스크를 이용해서 IP주소의 네트워크 주소를 구분하기 위해서 이다. 서브넷 마스크는 32비트 숫자로 구성된 IP주소이며, IP주소를 네트워크 주소와 호스트 주소로 나눈다. 서브넷 마스크는 32비트 숫자들 중 네트워크 주소를 모두 1로 표현하고 호스트 주소를 0으로 표기한다. 주어진 네트워크에서 두개의 호스느 주소는 특정 목적을 위해 예약되어있어서 호스트한테 할당될 수 없다. 0 인 주소는 네트워크 주소에 할당되어 있고, 255는 broadcast address이기 때문에 호스트에게 할당될 수 없다. – AND 연산 – 1 AND 1 = 1 1 AND 0 = 0 0 AND 1 = 0 0 AND 0 = 0

오버로딩(Overloading) 메서드도 변수와 마찬가지로 같은 클래스 내부에서 구별될 수 있어야 하기 때문에 각기 다른이름을 가져야한다. 그러나 자바에서는 한 클래스 내에 이미 사용하려는 이름과 동일한 이름을 가진 메서드가 있더라도 매개변수의 개수, 매개변수 타입이 다르면 다른 메서드로 간주한다. 이처럼, 한 클래스 내부에 같은 메서드 이름을 갖지만 다른 동작을 하는 메서드를 메서드 오버로딩, 혹은 오버로딩 이라고 한다. 오버로딩을 사용하기 위한 조건 오버로딩을 하기 위해서는 다음과 같은 조건을 만족 시켜야한다. 같은 클래스 내부에 동일 한 이름을 갖은 메서드가 존재 해야한다 매개변수의 개수 또는 타입이 달라야한다 1234567891011121314class People{ String name; int age; public People(String name, int age){ this.name = name; this.age = age; } public People(String name){ this.name = name; this.age = 18; }} 위 예제를 보면 People 이라는 클래스 내부에 두개의 People 디폴트 생성자 함수가 존재한다. 동일한 이름을 갖기 때문에 에러를 발생 시킬것 같지만, 둘은 다른 메서드 이다. 첫번째 생성자 함수는 두개의 매개변수(이름, 나이)를 입력해야 하지만 두번째 생성자 함수는 이름만입력을 하면 된다. 메소드 오버로딩을 사용하게 되면: 동일한 기능을 하는 메소드를 다른 이름으로 여러개를 만들 필요가 없다 메소드의 이름을 절약할 수 있다. 가변인자(varargs) 와 오버로딩 가변인자(variable arguments): 동적으로 매개변수의 개수를 지정. 가변인자는 타입...변수명 과 같은 형식으로 선언한다. public printStream printf(String format, Object... args) // 가변인자 사용방법
