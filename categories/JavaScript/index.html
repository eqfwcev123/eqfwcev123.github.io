2.variables

Javascript Variables during the variable declaration stage, a variable is created. More specifically, a variable allocates a memory space where it can store a value and connects the variable name with the address of the memory space. The memory space reserved during variable declaration is protected so that no one can use the space until it is released. In order to use a variable, a variable must be first declared. To declare a variable, var, let, const can be used. The Javascript engine performs variable declaration in two steps: Declaration phase: Registers a variable name to tell the Javascript engine that a variable is created Initialization phase: Free up memory space to store values and implicitly assign undefined Where are variable names stored? Variable names are stored in an areas called execution context. Execution context is an area where the Javascript engine provides necessary environment for evaluating and executing source code and actually manages the execution result of the code. Variable names and variable values are registered and managed as objects in the execution context. There are few differences when we use var, let, const When a variable is created using the var keyword, Javascript engine executes declaration phase and initialization phase simultaneously. For instance, if we declare a variable as var name;, a variable is first declared and it is initialized with a value undefined. Variable Hoisting 123console.log(score);var score; The code that references the variable precedes the variable declaratoin. Javascript code is executed sequentially, so console.log(name) is first implemented, followed by the code in the next line. Therefore, at the time when console.log(name) is executed, the declaration of a vriable score has not been executed so it can be expected that a Reference error could occur. However, rather than a reference error, undefined will be returned. The reason why undefined is returned is variable declarations are executed before console.log(name). To be more specific the variables declared using var keyword are declared and initialized before run-time when the source code is executed line by line.
