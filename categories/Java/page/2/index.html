inheritance

자바 상속(extends) 객체지향은 특징으로서 새로운 클래스를 정의 할 때 이미 구현된 클래스를 상속(inheritance) 받아서 속성이나 기능이 확장되는 클래스를 구현할 수 있다. 상속은 기존에 존재하는 클래스의 기능을 좀더 확장해서 사용하고 싶을때 사용한다 사용방법: 자식 클래스 extends 부모클래스`: 자식 클래스가 부모클래스한테서 상속을 받는다는 뜻이다. 다중 상속을 할 수 없다: 클래스C 가 클래스A, 클래스B한테서 상속을 받을 수 없다.(부모 클래스는 무조건 한명). 자바는 single inheritance 만 허용한다 상위 클래스는 하위 클래스보다 일반적인 개념과 기능을 갖는다 하위 클래스는 상위 클래스보다 구체적인 개념과 기능을 갖는다 생성자함수와 초기화 블럭(클래스 초기화 블럭/인스턴스 초기화 블럭)은 상속되지 않는다 자식 클래스의 멤버 개수는 조상 클래스보다 항상 같거나 많다 접근제어자가 Private 이거나 default일 멤버들은 상속되지 않는다기 보다는 상속은 받지만 자식 클래스가 접근을 할 수없는것이다. 두개의 클래스(자식 클래스 A 와 자식 클래스 B)가 한개의 부모 클래스 한테서 상속을 받더라도 자식 클래스 A 와 B는 아무런 관계가 없다. 상속 관계에서 형제 관계는 없다. 하위 클래스는 상위 클래스의 기능을 마음대로 사용할 수 있지만, private 한 변수와 메소드는 사용할 수 없다. 만약에 외부 클래스는 내 private 변수/메소드를 사용할 수 없지만 내 자식 클래스는 사용할 수 있게 하고 싶을 경우 private 이 아닌 protected 를 사용하면 된다. private 는 선언한 클래스 내부에서만 사용할 수 있다. 상속을 한다고 해도 사용할 수 없다. protected 를 사용해서 내 private 변수/메소드를 오직 내 자식 클래스들만 쓸수 있게 만들 수 있다. 예를들어 포유류 라는 부모클래스를 인간이 상속받는다: class Human extends Mammal{} 1// 고객관리 프로그램: 고객이라는 부모클래스 밑에 VIP고객 클래스와 다이아회원 클래스가 있다

Understanding Process, Thread and Data structures

Process A program in execution is often referred as process and a thread is a subset of a process. A process can either contain one thread (Single thread process) or mutiple threads(Multi-thread process). Inside a Process contains 4 different memory area: Data, Code, Stack, Heap Thread A unit of work that runs independently within a process. Specifically, a unit of work or task managed by the operating system. Threads and tasks (or tasks) can be used interchangeably. A single process is occured by JVM and statements within main()are a single thread. To create another thread other than main (), either inherit the Thread class or implement the Runnable interface. In multi-threaded operation, each thread can send and receive information to reduce errors in processing. Processes cannot send or receive information with each other.
