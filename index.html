다형성 Polymorphism

개체지향 개념에서 다형성이란 여러가지 형태를 가질 수 있는 능력을 의미하며, 자바에서는 한 타입의 참조변수로 여러 타입의 객체를 참조할 수 있도록 함으로써 다형성을 프로그램적으로 구현한다

조상 클래스 타입의 참조 변수로 자식 클래스의 인스턴스를 참조할 수 있다.

조상 클래스 타입의 참조변수로 자손 클래스의 인스턴스를 참조할 수 있다.

인스턴스를 다른 타입의 참조 변수로 참조

Human h = new Male()

Human 이라는 부모 클래스 타입의 참조변수는 자식 클래스인 Male 클래스의 인스턴스를 가리킨다. 이경우 실제 인스턴스가 Male 일 지더라도 h참조변수는 Human 클래스의 멤버만 사용할 수 있다. 이는 어떻게 보면 당연한 것이다.

h 타입의 참조 변수로는 자식 인스턴스 중에서 Human 클래스의 멤버들(isMale 자식 클래스한테 상속 한 멤버들)만 사용할 수 있다. 즉, h.isMale 와 h.isMale()을 할 수 없는 것이다.

자식 클래스가 부모 클래스 한테서 상속을 받게 되면 자식 클래스의 인스턴스는 부모 클래스의 모든 멤버(메소드 와 멤버 변수) 와 자기자신이 가지고있는 모든 멤버들을 갖게된다. 다시말해 자식 클래스의 인스턴스는 부모 클래스의 인스턴스보다 더많은 멤버 변수와 메소드를 갖고 있다는 것이다.

이제 Human h = new Male() 를 다시 보면 자식 클래스의 인스턴스를 생성 했는데, 참조변수의 타입은 부모 클래스인 것은 10리터의 물을 500ml 패트병으로 담으려고 하는 것이다. 500ml의 패트병으로는 500ml 치 의 물만 담을 수 있다. 마찬가지로 h 참조 변수로는 부모클래스타입이 저장할 수 있는 멤버변수/메소드만 저장할 수 있다.

다시 말해 인스턴스의 타입과 상관 없이 참조변수의 타입에 따라 해당 인스턴스가 가질수 있는 메버변수와 메소드의 개수가 달라진다.

자식 클래스 타입의 참조변수가 부모클래스의 인스턴스를 저장할 경우

isMale p = new Human()

위의 예제와 반대로 이번에는 부모클래스의 인스턴스를 자식 타입의 참조변수로 참조를 하고 있다.

자식 클래스 타입의 참조변수로 부모 클래스의 인스턴스를 참조할 수 없다. 그 이유는 실제 인스턴스인 Human()의 멤버 개수보다 참조변수 p가 사용할 수 잇는 메버 개수가 더 많기 때문이다.그래서 이를 허용하지 않는다.

isMale 자식 클래스에는 isMale,isMale() 멤버 들이있다. p 참조변수는 자식 클래스 이기 때무네 p 참조 변수로 p.isMale 혹은 p.isMale()을 사용 하려 할 수 있다. 하지만 p가 참조하고 있는 클래스에는 isMale 와 isMale()이 존재하지 않기 때문에 이를 사용할려고 하면 문제가 발생한다.

그렇기 때문에 자손타입의 참조변수로 부모타입은 인스턴스를 참조하는것은 존재 하지않는다. 한가지 기억 해야 하는것은 참조 변수가 사용할 수 있는 멤버의 개수는 인스턴스의 멤버 개수보다 같거나 적어야 한다.(더많으면 안된다)

​ 클래스는 상속을 통해서 확장 될 수 있더라도 축소될 수는 없어서 부모 클래스 인스턴스의 멤버 변수 개수는 자손 클래스 인스턴스의 멤버보다 항상 적거나 같아야 한다. –> 자식 클래스타입의 참조변수에 부모 클래스의 인스턴스를 넣을 수 없다

참조변수의 타입이 참조변수가 참조하고 있는 인스턴스에서 사용할 수 있는 멤버의 개수를 결정한단든 사실을 꼭 알고 있어야 한다.

​ 조상 타입의 참조변수로 자손 타입의 인스턴스를 참조할 수 있다.

반대로 자손 타입의 참조변수로 조상 타입의 인스터스를 참조할 수 없다.

참조 변수의 다형성

기본형 변수와 마찬가지로 참조형 변수도 형변환을 할 수 있다. 단, 서로 상속 관계에 있는 클래스 사이에서만 형변환을 할 수 있다.(부모 클래스 타입과 자식 클래스 타입 끼리만 형변환을 할 수 있다).

조상 타입의 참조변수를 자식 클래스 타입의 참조변수로 형변환 하는것을 다운 캐스팅, 그 반대를 업캐스팅 이라고 한다.

부모클래스 에서 자식클래스로 형변환: Down Casting 형변환 생략 불가

자식 클래스에서 부모 클래스로 형변환: Up Casting 형변환 생략 가능

부모 클래스에서 자식 클래스로 다운 캐스팅을 할때는 생략을 해줘야 하지만 자식 클래스에서 부모클래스로 현변환을 할 때는 암묵적으로 자동 형변환이 가능 하다. 그 이유는 자식 클래스의 데이터 타입이 더 많은 데이터를 가지고 있기 때문에 형변환을 해줘야 한다. 쉽게 생각하면 5리터 물통의 물을 500미리 패트병에 넣을려면 물을 버리고 넣어야 한다. 하지만 500ml의 물을 5리터 물통에 넣을 때는 그냥 넣을 수 있다.

자바에는 형재 관계라는 것이 존재 하지 않고 부모 자식 관계만 있다. 그렇기 때문에 형재 관계끼리 타입 변환을 할 수 없다(위 예제일 경우 엠뷸런스와 소방차 인스턴끼리의 타입변환이 불가능 한 것이다).

형변환은 참조변수의 타입을 변환하는 것이지 인스턴스를 변환하는 것이 아니다. 그렇기 때문에 참조변수의 형변환은 인스턴스에 아무런 영향을 주지 않는다.

자식 클래스의 인스턴스 변수를 부모클래스의 타입으로 형변환을 할 경우 전에 배운 다형에서 처럼 자식 클래스는 자식 클래스가 기존에 갖고 있던 자기 자신의 멤버 변수/ 메소드를 사용할 수 없게 된다. 위 예제로 따지면 fe 는 다운캐스팅 되어서 car 참조변수에 저장이 된다. 이제 car 참조변수는 water()메소드를 사용할 수 없다.

서로 상속관계에 있는 클래스 타입의 참조변수간의 형변환은 양방향으로 자유롭게 할 수 있으나, 참조변수가 참조하고 있는 인스턴스의 자손타입으로 형변환을 하는것은 허용되지 않는다. 참조변수가 가리키는 인스턴스의 타입이 무엇인지 확인 하는 것이 중요하다

인스턴스 instance of(클래스)를 사용하면 해당 인스턴스가 부모 클래스의 인스턴스인지 알 수 있다.

참조 변수와 인스턴스의 연결

부모 타입의 참조변수와 자식 타입의 참조변수의 차이는 사용할 수 있는 멤버 변수와 메소드의 차이를 제외하고 한가지가 더있다.

조상 클래스에 선언된 멤버변수와 같은 이름의 인스턴스 변수를 자손 자식 클래스에 중복으로 정의했을 때 조상 타입의 참조변수로 자식인스턴스를 참조하는 경우와 자식 타입의 참조변수로 자식 인스턴스를 참조하는경우 다른 결과가 나온다.

메서드의 경우 조상 클래스의 메서드를 자손의 클래스에서 오버라이딩 한 경우에도 참조 변수의 타입에 관계없이 항상 실제 인스턴스의 메서드(오버라이딩된 메서드)가 호출되지만, 멤버변수의 경우 참조변수의 타입에 따라 달라진다.

멤버 변수가 조상 클래스와 자속 클래스에 중복으로 정의된 경우, 조상 타입의 참조변수를 사용했을 때는 조상 클래스에 선언된 멤버가 사용되고, 자손 타입의 참조변수를 사용하면 자식 클래스에 선언되 멤버변수가 사용된다. 다시말해, 참좁변 수의 타입에 따라 다른 멤버변수가 호출이 된다.

결론: 상속을 받은 자식 클래스가 자기 자신의 메소드와 인스턴스 멤버를 가지고 있으면 자기것을 사용하고, 없으면 부모클래스의 멤벼 변수와 메소드를 빌려온다.

메소드는 만약에 오버라이딩 할 경우 부모의 메서드를 덮어 씌운거라고 생각하면 된다. 그렇기 때문에 참조변수의 타입과 상관 없이 호출을 하게되면 부모의 메서드가 아닌 자식 클래스에서 오버라이딩 한 자식클래스의 메서드가 호출 된다.

제어자 (Modifier)

제어자(Modifier)는 클래스, 변수 또는 메서드의 선언부에 함께 사용되어 부가적인 의미를 부여한다. 제어자의 종류는 크게 접근 제어자와 그 외의 제어자로 나눌 수 있다.

접근 제어자 - public, protected, default, private

그외 - static, final, abstract, native, transient, synchronized, volatile, strictfp

제어자는 클래스나 멤버 변수와 메서드에 주로 사용되며, 하나의 대상에 대해서 여러 제어자를 조합하여 사용하는 것이 가능하다. 단, 접근 제어자는 한 번에 네 가지 중 하나만 선택해서 사용할 수 있다. 즉, 하나의 대상에 대해서 public, private 을 함께 사용할 수 없다.

static

인스턴스 변수는 하나의 클래스로부터 생성되었더라도 각기 다른 값을 유지하지만, 클래스 변수(static 변수)는 인스턴스에 관계 없이 같은 값을 갖는다. 그 이유는 하나의 변수를 모든 인스턴스가 공유하기 때문이다.

static이 붙은 멤버 변수와 메서드, 그리고 초기화 블럭은 인스턴스가 아닌 클래스에 관계된 것이기 때문에 인스턴스를 생성하지 않고도 사용할 수 있다.

인스턴스 메서드와 정적 메서드의 근본적인 차이는 메서드 내에서 인스턴스 멤버를 사용하는 가의 여부에 있다.

Package

패키지는 클래스의 묶음이다. 패키지에는 클래스 또는 인터페이스를 포함시킬 수 있으며, 서로 관련된 클래스들 끼리 그룹 단위로 묶어 놓음으로써 클래스를 효율적으로 관리할 수 있다. 같은 이름의 클래스 일지라도 서로 다른 패키지에 존재하는 것이 간으하므로, 자신만의 패키지 체계를 유지함으로써 다른 개발자가 개발한 클래스 라이브러리의 클래스와 이름이 충돌하는것을 방지할 수 있다.

클래스의 실제 이름은 클래스의 패키지 이름을 포함한 이름이다.

클래스가 물리적으로 하나의 .class 파일 인것과 마찬가지로, 패키지는 물리적으로 하나의 디렉토리이다. 그렇기 때문에 특정 패키지에 존재하는 하나의 클래스는 패키지 라는 디렉토리에 존재하는 클래스라고 생각하면 된다.

java.lang.String이라는 클래스는 java의 sub 디렉토리인 lang에 속하는 String.class 인 것이다. 다시말해 java 라는 패키지 내부에 lang 이라는 패키지가 있고, 그안에 String.class 가 있는 것이다.

package 패키지 이름 을 이용해서 패지를 선언할 수 있다.

import 문

모든 소스 파일에서 import 문은 package 다음 문에, 그리고 클래스 선언 문 이전에 위치해야 한다. 그리고 import문은 package 문과 달리 한 소스파일에 여러번 선언할 수 있다.

import 패키지명.클래스명 를 이용해서 import를 할 수 있다.

각 클래스 마다 static 한 멤버들이 있는데, 이런 정적 멤버들을 import하기 위해선느 import static 패키지.클래스명을 해줘야 한다.

오버라이딩

조상 클래스로부터 상속받은 메서드의 내용을 변경하는 것을 오버라이딩 이라고 한다.

부모 클래스한테서 상속 받은 메소드를 그대로 사용해도 되지만 자식 클래스 자신한테 맞춰서 메소드를 변경할 수 있다. 메소드를 오버라이딩 한다고 해서 다른 클래스들이 부모 클래스한테서 상속 받는 메소드도 변경 되는 것은 아니다.

Point3D 클래스는 Point 클래스를 상속 받는다. 하지만 Point3D 클래스 3Dimension 좌표를 반환 해야 하기 때문에 부모한테서 상속받은 메소드를 사용하면 안된다. 이때 부모한테서 상속받은 메소드를 오버라이딩 하면 된다. 상속받은 getLocation()메소드를 오버라이딩 해서 자기 자신의 새로운 메소드로 사용을 하면 x,y,z 세개의 좌표를 반환받을 수 있다.

오버로딩(Overloading)

메서드도 변수와 마찬가지로 같은 클래스 내부에서 구별될 수 있어야 하기 때문에 각기 다른이름을 가져야한다. 그러나 자바에서는 한 클래스 내에 이미 사용하려는 이름과 동일한 이름을 가진 메서드가 있더라도 매개변수의 개수, 매개변수 타입이 다르면 다른 메서드로 간주한다.

이처럼, 한 클래스 내부에 같은 메서드 이름을 갖지만 다른 동작을 하는 메서드를 메서드 오버로딩, 혹은 오버로딩 이라고 한다.

오버로딩을 사용하기 위한 조건

오버로딩을 하기 위해서는 다음과 같은 조건을 만족 시켜야한다.

  1. 같은 클래스 내부에 동일 한 이름을 갖은 메서드가 존재 해야한다
  2. 매개변수의 개수 또는 타입이 달라야한다

위 예제를 보면 People 이라는 클래스 내부에 두개의 People 디폴트 생성자 함수가 존재한다. 동일한 이름을 갖기 때문에 에러를 발생 시킬것 같지만, 둘은 다른 메서드 이다. 첫번째 생성자 함수는 두개의 매개변수(이름, 나이)를 입력해야 하지만 두번째 생성자 함수는 이름만입력을 하면 된다.

메소드 오버로딩을 사용하게 되면:

  1. 동일한 기능을 하는 메소드를 다른 이름으로 여러개를 만들 필요가 없다
  2. 메소드의 이름을 절약할 수 있다.

클래스 메서드(static 메서드) 와 인스턴스 메서드

메서드 앞에 static 일 들어가면 해당 메서드는 static 메서드 이고 붙어 있지 않으면 인스턴스 메서드 이다. 클래스 메서드도 클래스변수처럼 객체를 생성하지 않고 ‘클래스이름.클래스메서드( )’ 형식으로 사용한다. 인스턴스 메서드는 인스턴스 변수와 마찬가지로 인스턴스를 생성해야 인스턴스 메서드를 ‘인스턴스.인스턴스메서드( )’ 형식으로 사용할 수 있다.

인스턴스 메서드는 인스턴스 변수와 관련된 작업을 하는, 즉 메서드의 작업을 수행하는 데 인스턴스를 필요로 하는 메서드 이다. 그런데 인스턴스 변수는 인스턴스를 생성해야하만 만들어 지므로, 인스턴스 메서드 역시 인스턴스를 생성해야만 호출 해야한다.

반면에 메서드 중에서 인스턴스와 관계없는 메서드를 클래스 메서드(정적 메서드) 로 정의한다.

  • 클래스를 설계할 때, 멤버변수 중 모든 인스턴스에 공통적으로 사용해야하는 것에 static을 붙인다.

    • 생성된 각 인스턴스는 서로 독립적이기 때문에 각 인스턴스의 변수는 서로 다른 값을 유지한다. 그러나 모든 인스턴스에서 같은 값이 유지되어야 하는 변수는 static 으로 선언을 하면 된다.

    • 클래스 변수(static 변수) 는 인스턴스를 생성하지 않아도 사용할 수 있다.

      • static이 붙은 변수는 클래스가 메모리에 올라갈 때 이미 자동적으로 생성되기 때문이다.

    • 클래스 메서드는 인스턴스 변수를 사용할 수 없다.

      • 인스턴스 변수는 인스턴스가 반드시 존재햐아만 사용할 수 있다. 클래스 메서드는 인스턴스 생성 없이 호출 가능하므로 클래스 메서드가 호출되었을 때 인스턴스가 존재하지 않을 수 있다. 그래서 클래스 메서드에서 인스턴스 변수의 사용을 금지한다. 반면에 인스턴스 변수나 인스턴스 메서드에서는 static이 붙은 멤버들을 사용하는 것이 언제나 가능하다. 인스턴스 변수가 존재한다는 것은 static 변수가 이미 메모리에 존재한다는 것을 의미한다.

        ​ 앞에서 말한것 처럼 클래스 파일이 사용이 되면 JVM 은 해당 .class 파일을 읽어서 분석하여 클래스에 대한 정보를 메소드 영역에 저장을 한다. 이때 클래스의 정적 메소드와 정적 멤버 변수가 최 우선적으로 생성이 된다.

    • 메서드 내에서 인스턴스 변수를 사용하지 않는다면 static을 붙이는 것을 고려한다.

      • 메서드의 작업내용 중에서 인스턴스 변수를 필요로 한다면, static을 붙일 수 없다. 반대로 인스턴스 변수를 필요로 하지 않는다면 static을 붙이는 것이 좋다. 메서드 호출 시간이 짧아지므로 성능이 향상된다. static을 안붙인 메서드(인스턴스 메서드)는 실행시 호출되어야할 메서드를 찾는 과정이 추가적으로 필요하기 때뭉네 시간이 더 걸린다.
