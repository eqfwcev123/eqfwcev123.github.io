Credentials 파일에 있는 SecretsManager 사용하기

  1. 세션으로 불러오기.

    boto3 를 이요하게 되면 boto3 가 자동으로 세션 관리를 해주지만, 사용자가 세션을 직접 관리를 할 수 있다.

  1. session 으로 aws 서비스 사용하기

    AWS SecretsManager에 사용자의 Secret key를 저장해놓고, 해당 Secret Key를 받아와서 사용하고싶을 경우 SecretsManager서비스를 사용해야한다.

  2. SecretsManager 내부에 있는 secret key 받아오기.

    Secret Key를 받아올려면 get_secret_value() 함수를 사용해야한다.

  3. 받아온 secret_key 내부의 SecretsString 을 사용하면 된다. 하지만 받아올 경우 Json String 형식으로 받아와 지기 때문에 파이썬 객체로 변환 해줘야 한다. json.loads() 를 이용해서 변환을 시켜주면 된다.

운영체제

Posted on 2020-02-09 Edited on 2020-02-10 In Computer Science

운영체제의 역할: 시스템 자원(System Resource) 관리자

운영체제의 역할: 시스템 자원(System Resource) 관리자

Operating System 또는 OS 라고 부른다

여기서 시스템 자원이란? 컴퓨터 하드웨어

–> CPU(중앙처리장지), Memory(DRAM, RAM)

–> I/O Devices(입출력 장치) : Monitor, Mouse, Key Board etc

–> 저장매체: SSD, HDD(하드디스크)

​ 컴퓨터 하드웨어는 스스로 할수 있는 것이 하나도 없다

운영체제가 해주는 역할들:

  1. CPU: 각 프로그램이 얼마나 CPU를 사용할 지를 결정할 수 없다.
  2. Memory: 각 프로그램이 어느 주소에 저장되어야 하는지, 어느정도의 메모리 공간을 확보 해줘야 하는지를 결정할 수 없다.
  3. 저장매체(HDD, SSD): 어떻게, 어디에 저장할지는 결정할 수 없다
  4. I/O (키보드/마우스): 키보드를 입력하면 어떤 단어가 나와야 하고, 마우스를 움직이면 마우스 포인터가 어느정도 움직이는지를 운영체제가 결정 해준다

boto3 요약

Posted on 2020-02-01 In Server Deploy

Boto3 Credentials

Boto 는 다양한 방법으로 환경설정을 할 수있다. 그중 AWS Credentials 와 region은 꼭 가지고 있어야 한다.

Boto3설정을 할때 두가지 데이터가 존재한다: credential data 와 non-credential data 두가지이다.

Credential data는 aws_access_key_id, aws_secret_access_key, aws_session_token 와 같은 데이터가 있고, non-credential data 에는 region 정보가 있다.

Credential data 와 non Credential데이터에 의해 검색 과정이 달라질 수 있기 때문에 두가지 데이터를 잘 구분할 줄 알아야 한다. Boto3는 non-credenital data를 찾을때와는 다르게 credential data를 찾을때는 다양한 위치에서 credential data를 찾는다.

boto3가 credential 데이터를 찾을때 credential data가 있을만한곳을 모두 검색을 하는데, 검색을 하는 순서는 다음과 같다:

  1. boto.client() 메서드로 전달된 매개변수에 있는 credential 데이터
  2. Session 객체를 생성할 때 credential을 매개변수로 전달
  3. 환경 변수
  4. 공유되는 credential 파일 (~/.aws/credentials)
  5. AWS config파일(~/.aws/config)
  6. Assume Role Provider
  7. Boto2 config file(/etc/boto.cfg and ~/.boto)
  8. Amazon EC2(IAM role 보유)에 있는 인스턴스 Metadata 서비스

1. Client(), Session()메소드의 파라미터로 credential data 전달

client 혹은 session을 생성할 때 매개변수로 credential data를 전달하는 방법이다.

ACCESS_KEY, SECRET_KEY_SESSION_TOKEN 에 사용자의 접근키를 그대로 입력하는 방법을 추천하지 않는다.

docker 이미지 레이어(Docker Image Layer)

Posted on 2020-01-30 In Docker

Docker Image Layer의 구조

Docker 의 이미지를 이용해서 docker run 을 하면 Docker 는 도커가 관리하는 파일 시스템 영역에 이미지를 복사한다. 복사후 docker는 이미지의 최상단에 컨테이너 레이어 라고 불리는 하나의 얇은 레이어를 추가하여 컨테이너를 생성한다. 그리고 사용자에게 유니온 파일 시스템을 이용하여 마치 이러한 여러개의 파일 시스템(Image layer)으로 구성되어 있는 이미지 스택 구조가 하나의 파일 시스템 처럼 보이도록 하게 한다.

설명: Docker 이미지가 기본적으로 가지고 있는 이미지 레이어들

설명: Docker run [이미지] 를 통해 이미지로 컨테이너를 생성할 경우 기존의 이미지 레이어들 위에 conatiner layer 가 생성된다.

사용자가 컨테이너 안에서 읽고 쓰는 모든 작업 들은 이 컨테이너 레이어에 기록되고 이미지 레이어에는 적용되지 않는다. 쉽게 말해 Image layer는 변경 불가하고(read only layer), Container layer는 변경 가능하다(Readable/Writable layer).

다른 사용자가 같은 이미지를 이용해서 container 를 실행할 경우 다른 Image layer만 생성이 되지, 다른 사용자들이 생성한 Container layer는 복사되지 않는다.

이미지 레이어는 삭제 되지 않지만 컨테이너 레이어는 해당 컨테이너가 종료될경우 같이 소멸된다. 만약에 사용자가 Container Layer에서 작업하던 내용을 Image layer로 적용 하고 싶을 경우 docker commit 을 사용하면 된다. docker commit 을 사용하게 되면 기존의 Image layer 들과 직접 작성한 Container Layer를 이용해 새로운 이미지를 생성할 수 있다.

Docker image를 생성할 수 있는 두가지 방법:

  1. docker build 를 이용해서 설정한 Dockerfile 에 맞는 이미지 생성
  2. 컨테이너에서 작성한 container layer 와 image layer를 합해서 새로운 image 생성

우분투에서 도커 설치/삭제

Posted on 2020-01-29 In Docker

우분투에 Docker 설치하기

sudo apt-get remove docker docker-engine docker.io containerd runc : 이전 Docker 버전 삭제

Docker-CE 를 처음 설치할 경우 Docker repository 를 설정 해줘야 한다. 레포를 설정 한 이후 Docker 를 설치/업데이트 할 수 있다.

  1. sudo apt-get update : apt 패키지 업데이트

  2. sudo apt-get install \
    apt-transport-https \
    ca-certificates \
    curl \
    gnupg-agent \
    software-properties-common

# apt가 HTTPS를 통해 저장소를 사용할 수 있도록 패키지를 설치해준다

Docker 설치

  1. sudo apt-get update
  2. sudo apt-get install docker-ce docker-ce-cli containerd.io
  3. sudo docker run hello-world : docker 가 설치 되었는지 확인하기

Docker 제거

sudo apt-get purge docker-ce : Docker CE 를 삭제한다

sudo rm -rf /var/lib/docker: Docker 의 이미지, 컨테이너 등은 자동으로 삭제가 되지 않기 때문에 직접 삭제를 해줘야 한다.
