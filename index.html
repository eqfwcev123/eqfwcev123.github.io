링크드 리스트

링크드 리스트 구조

  • 연결 리스트 라고도 한다
  • 배열은 순차적으로 연결된 공간에 데이터를 나열 하는 데이터 구조이다
  • 링크드 리스트는 떨어진 곳에 존재하는 데이터를 화살표로 연결해서 관리하는 구조이다
  • 본래 C언어에서는 주요한 데이터 구조이지만, 파이썬은 링크드리스트 타입이 링크드리스트의 모든 기능을 지원한다
  • 배열의 단점: 미리 연결된 공간을 예약(확보 해야한다)해야한다. 연결되지 않을경우 배열로서의 기능을 할 수 없다
  • 이러한 문제를 해결하기 위해 링크드리스트를 사용한다. 링크드 리스트는 미리 공간을 확보해둘 필요가 없이 필요할때 데이터를 추가해서 사용하면 된다.
  • 데이터들의 메모리 주소가 순차적으로 나열 되어있을 필요가 없다. 포인터가 다음 데이터의 주소를 가리키기 때문이다.

Stack 자료구조

  • 데이터를 제한적으로 접근할 수 있는 자료구조
    • 한쪽 끝에서만 자료를 넣거나 뺄 수 있는 구조

  • 스택은 LIFO(Last in First Out) 데이터 관리 방식을 따른다
  • 대표적인 스택의 활용 내용
    • 컴퓨터 내부의 프로레스 구조의 함수 동작 방식
  • 주요기능
    • push(): 데이터를 스택에 넣기
    • pop(): 데이터를 스택에서 꺼내기

Queue 자료구조

Posted on 2020-01-06 In Algorithm

자료구조 Data Structure

  • 가장 먼저 넣은 데이터를 가장 먼저 꺼낼 수 있는 구조

    • 음식점에서 가장 먼저 줄을 선 사람이 제일 먼저 음식점에 입장하는 것과 동일하다
    • FIFO 방식으로 스택과 동작하는 방식이 상반된다.

  • 알아 둬야할 용어들

    • Enqueue: 큐에 데이터를 넣는 기능
    • Dequeue: 큐에서 데이터를 꺼내는 기능

  • 파이썬 Queue 라이브러리를 활용해서 큐 자료 구조 사용하기

    • queue 라이브러리 에는 다양한 큐 구조로 Queue(), LifoQueue(),PriorityQueue() 메서드를 제공

    • 프로그램을 작성할 때 프로그램에 따라 적합한 자료구조를 사용

      • Queue(): 가장 일반적인 큐 자료구조
      • LifoQueue(): 나중에 입력된 데이터가 먼저 출력되는 구조(스택 구조라고 보면된다)
      • PriorityQueue(): 데이터마다 우선 순위를 넣어서, 우선순위가 높은 순으로 데이터 출력

      일반적인 큐 외에 다양한 정책에 적용되는 큐들이 있다.

Understanding Process, Thread and Data structures

Posted on 2020-01-05 Edited on 2020-01-06 In Java

Process

  • A program in execution is often referred as process and a thread is a subset of a process.

  • A process can either contain one thread (Single thread process) or mutiple threads(Multi-thread process).

  • Inside a Process contains 4 different memory area: Data, Code, Stack, Heap

Thread

A unit of work that runs independently within a process. Specifically, a unit of work or task managed by the operating system. Threads and tasks (or tasks) can be used interchangeably.

  1. A single process is occured by JVM and statements within main()are a single thread.
  2. To create another thread other than main (), either inherit the Thread class or implement the Runnable interface.
  3. In multi-threaded operation, each thread can send and receive information to reduce errors in processing.
  4. Processes cannot send or receive information with each other.

JavaScript Study

Prototype

How porototype works in JavaScript:

A Person constructor function is created during function declaration (Syntax analysis stage) and when the function is created, the prototype of the function is also created. Rather than storing isHuman inside the function, the isHuman property is stored inside Person.prototype and is shared by all its instances.
To create an instance of Person constructor function, the new keyword is used and 'Lee','Kim' are passed as arguments to the constructor function. Before using the new keyword Javascript engine always checks whether the function is a constructor or not.
