서버 배포

  1. 보안그룹 생성 : 보안그룹 생성 > 보안그룹 이름/설명/인바운드 입력

  2. 인스턴스 생성 : Ubuntu 18.04 생성 > 보안 그룹 구성에서 기존의 보안그룹/새로운 보안 그룹 선택

  3. 인스턴스를 생성하면서 Secret 키를 생성할 수 있는데, 생성을 하면 ~/.ssh 디렉토리에 저장. AWS에서 인스터스를 생성 하면 키만 받고, 자물쇠는 우리가 인스턴스를 생성할 때 AWS 에서 자물쇠를 넣어준다.(다시말해 우리는 열쇠만 가지고 있는것이다.)

  4. EC2 서버로 접근하기: ssh -i ~/.ssh/[보안키이름].pem [유저]@[서버 주소]. 이렇게 하면 원래는 서버로 접속을 할 수 있지만 처음 설정 할때는 접근할 수 없다. EC2에 인바운드와 아웃바운드 설정을 해줘야 한다. 인바운드 설정에 유형이라는 항목이 있는데, 항목 내부에 다양한 선택지들이 있다. 그중 SSH를 설정해야 하는데, SSH를 설정하는 이유는 사용자가 SSH를 이용해서 인스턴스에 접근을 해야하기 때문이다.

    • 인바운드 : 외부에서 인스턴스로 들어오는 요청이다
    • 아웃바운드 : 인스턴스에서 외부로 나가는 트래픽이다. 0.0.0.0 은 어디든 갈 수 있다

  5. 인바운드를 설정하고 나서 다시 서버에 접근을 할려고하면 WARNING: UNPROTECTED PRIVATE KEY 라고 나온다. 그 이유는 우리가 사용하고 하는 secret키에 대한 보안이 너무 취약하기 때문이다. ~/.ssh 파일에서 ls-al을 해보면 -rw-|r--|r--@ 라고 나온다.

    • 맨앞 - 에 d가 있으면 디렉토리라는 뜻이다.
    • r 는 read 를 read를 의미한다
    • w 는 write 를 의미한다
    • x 는 executable을 의미한다
    • u 는 user(사용자)를 의미한다
    • g 는 그룹을 의미한다
    • o 는 기타 를 의미한다.
    • (파일타입|user|group|other) || 링크수 || 소유자 || 그룹 || 파일크기 || 파일수정시간 || 파일이름

  6. chmod와 숫자를 이용해서 해당 파일의 권한 변경하기: 숫자를 이용해서 해당파일의 user|group|other의 권한을 한번에 변경할 수 있다.

    • rwx 가 있는데 r 은 2^2, w 는 2^1, x 는 2^0을 의미한다. 다시말해 rwx = 2^2 + 2^1 + 2^0 = 7 을 의미한다.
    • r-x 가 있는데 r 은 2^2, x 는 2^0을 의미한다. 다시말해 rwx = 2^2 + 2^0 = 5 을 의미한다.
    • 쉽게 말해 r 는 4, w 는 2, x 는 1을 의미한다.
    • 권한을 부여할때 다음과 같이 부여한다. chmod 777 a.py. 이는 즉 a.py파일에 rwx|rwx|rwx를 부여한는 것이다.
    • chmod 124 b.py 는 b.py파일에 –x|-w-|r– 권한을 부여하는것이다

추상 클래스

추상 클래스는 미완성 클래스라고 생각하면 된다. 클래스가 미완성 이라는 것은 멤버의 개수에 관계된 것이 아니라, 단지 미완성 메서드를 포함하고 있는 클래스 인것이다.

미완성 클래스로 완성된 제품을 만들 수 없이 추상클래스로 인스턴스를 생성할 수 없다. 추상클래슨은 상속을 통해서 자손클래스에 의해서만 완성될 수 있다.( 장고에서 class Meta: abstract = True 를 생각하면 된다. )

추상 클래스 자체로는 클래스로서의 역할을 다 못하지만, 새로운 클래스르 작성하는데 있어서 바탕이 되는 부모클래스 이다. 자식 클래스들이 부모클래스 한테서 상속을 받는 이유는 부모의 기능을 그대로 사용하기 위해서이다. 하지만 만약에 부모 클래스로 인스턴스를 생성할 필요가 없다고 치면 그때 추상 클래스를 사용하면 된다.

추상 클래스는 abstract를 클래스 앞에 붙여서 사용한다. 추상 클래스는 추상 메서드를 포함 하고 있다는것을 제외하고는 일반클래스와 다른점이 없다. 추상 클래스 에도 생성자가 있으며, 멤버변수와 메서드를 모두 사용할 수 있다.

추상 메서드

메서드의 구현부가 비어있을 경우 추상 메서드 라고 한다. 메서드를 미완성으로 보류하는 이유는 메서드의 내용이 상속받는 클래스에 따라 달라질 수 있기 때문에 조상 클래스에 메소드의 구현부를 비워두고, 나중에 자식 클래스가 추상 메서드를 상속 받아서 채워가는 형식이다.(모든 인스턴스들이 동일한 메서드를 가져야 하는데 부모 클래스 입장에서 당장 메서드를 사용할 수 없을 경우 비워두는 것이다.)

추상 클래스로 부터 상속받는 자손 클래스는 오버라이딩을 통해 조상인 추상클래스의 추상메서드를 모두 구현해주어야 한다. 만일 조상으로부터 추상 메서드 중 하나라도 구현하지 않는다면, 자손 클래스 역시 추상 메서드로 지정해줘야한다(다음 자식 이 상속 받아서 지정하고 사용)

다형성 Polymorphism

개체지향 개념에서 다형성이란 여러가지 형태를 가질 수 있는 능력을 의미하며, 자바에서는 한 타입의 참조변수로 여러 타입의 객체를 참조할 수 있도록 함으로써 다형성을 프로그램적으로 구현한다

조상 클래스 타입의 참조 변수로 자식 클래스의 인스턴스를 참조할 수 있다.

조상 클래스 타입의 참조변수로 자손 클래스의 인스턴스를 참조할 수 있다.

인스턴스를 다른 타입의 참조 변수로 참조

Human h = new Male()

Human 이라는 부모 클래스 타입의 참조변수는 자식 클래스인 Male 클래스의 인스턴스를 가리킨다. 이경우 실제 인스턴스가 Male 일 지더라도 h참조변수는 Human 클래스의 멤버만 사용할 수 있다. 이는 어떻게 보면 당연한 것이다.

h 타입의 참조 변수로는 자식 인스턴스 중에서 Human 클래스의 멤버들(isMale 자식 클래스한테 상속 한 멤버들)만 사용할 수 있다. 즉, h.isMale 와 h.isMale()을 할 수 없는 것이다.

자식 클래스가 부모 클래스 한테서 상속을 받게 되면 자식 클래스의 인스턴스는 부모 클래스의 모든 멤버(메소드 와 멤버 변수) 와 자기자신이 가지고있는 모든 멤버들을 갖게된다. 다시말해 자식 클래스의 인스턴스는 부모 클래스의 인스턴스보다 더많은 멤버 변수와 메소드를 갖고 있다는 것이다.

이제 Human h = new Male() 를 다시 보면 자식 클래스의 인스턴스를 생성 했는데, 참조변수의 타입은 부모 클래스인 것은 10리터의 물을 500ml 패트병으로 담으려고 하는 것이다. 500ml의 패트병으로는 500ml 치 의 물만 담을 수 있다. 마찬가지로 h 참조 변수로는 부모클래스타입이 저장할 수 있는 멤버변수/메소드만 저장할 수 있다.

다시 말해 인스턴스의 타입과 상관 없이 참조변수의 타입에 따라 해당 인스턴스가 가질수 있는 메버변수와 메소드의 개수가 달라진다.

제어자 (Modifier)

제어자(Modifier)는 클래스, 변수 또는 메서드의 선언부에 함께 사용되어 부가적인 의미를 부여한다. 제어자의 종류는 크게 접근 제어자와 그 외의 제어자로 나눌 수 있다.

접근 제어자 - public, protected, default, private

그외 - static, final, abstract, native, transient, synchronized, volatile, strictfp

제어자는 클래스나 멤버 변수와 메서드에 주로 사용되며, 하나의 대상에 대해서 여러 제어자를 조합하여 사용하는 것이 가능하다. 단, 접근 제어자는 한 번에 네 가지 중 하나만 선택해서 사용할 수 있다. 즉, 하나의 대상에 대해서 public, private 을 함께 사용할 수 없다.

static

인스턴스 변수는 하나의 클래스로부터 생성되었더라도 각기 다른 값을 유지하지만, 클래스 변수(static 변수)는 인스턴스에 관계 없이 같은 값을 갖는다. 그 이유는 하나의 변수를 모든 인스턴스가 공유하기 때문이다.

static이 붙은 멤버 변수와 메서드, 그리고 초기화 블럭은 인스턴스가 아닌 클래스에 관계된 것이기 때문에 인스턴스를 생성하지 않고도 사용할 수 있다.

인스턴스 메서드와 정적 메서드의 근본적인 차이는 메서드 내에서 인스턴스 멤버를 사용하는 가의 여부에 있다.

staic이 사용될 수 있는곳 - 멤버변수, 메스드, 초기화 블럭

Package

패키지는 클래스의 묶음이다. 패키지에는 클래스 또는 인터페이스를 포함시킬 수 있으며, 서로 관련된 클래스들 끼리 그룹 단위로 묶어 놓음으로써 클래스를 효율적으로 관리할 수 있다. 같은 이름의 클래스 일지라도 서로 다른 패키지에 존재하는 것이 간으하므로, 자신만의 패키지 체계를 유지함으로써 다른 개발자가 개발한 클래스 라이브러리의 클래스와 이름이 충돌하는것을 방지할 수 있다.

클래스의 실제 이름은 클래스의 패키지 이름을 포함한 이름이다.

클래스가 물리적으로 하나의 .class 파일 인것과 마찬가지로, 패키지는 물리적으로 하나의 디렉토리이다. 그렇기 때문에 특정 패키지에 존재하는 하나의 클래스는 패키지 라는 디렉토리에 존재하는 클래스라고 생각하면 된다.

java.lang.String이라는 클래스는 java의 sub 디렉토리인 lang에 속하는 String.class 인 것이다. 다시말해 java 라는 패키지 내부에 lang 이라는 패키지가 있고, 그안에 String.class 가 있는 것이다.

package 패키지 이름 을 이용해서 패지를 선언할 수 있다.

import 문

모든 소스 파일에서 import 문은 package 다음 문에, 그리고 클래스 선언 문 이전에 위치해야 한다. 그리고 import문은 package 문과 달리 한 소스파일에 여러번 선언할 수 있다.

import 패키지명.클래스명 를 이용해서 import를 할 수 있다.

각 클래스 마다 static 한 멤버들이 있는데, 이런 정적 멤버들을 import하기 위해선느 import static 패키지.클래스명을 해줘야 한다.
