Passport.js 사용법

인증 방법: passport.authenticate() 와 어떤 strategy를 이용할 것인지 적어줘야한다. Strategy 는 route를 사용하기전에 미리 설정 해줘야 한다.

인증 실패시 passport는 401 unauthorized를 반환하고 router 핸들러는 더이상 실행되지 않는다.

인증 성공시, 그다음 route 핸들러가 실행되고 req.user에는 인증된 사용자정보가 저장된다.

Configuration

Passport의 인증을 이용하기 위해서 3가지 설정을 한다.

  1. Authentication Strategies
  2. Application Middleware
  3. Sessions(optional)
  1. Strategies

Passport는 strategies를 이용해서 요청을 인증한다. Strategies에는 다양한 종류가 있는데, 사용자의 유저네임과 비밀번호를 인증하는 방식이거나 대리 인증방식인 OAuth 혹은 다양한 방식을 이요할 수 있다.

Passport한테 요청 인증을 하기 전에, 애플리케이션에서 사용할 strategy를 설정 해줘야한다.

strategy를 인증하는 방법은 use() 함수를 이용해서 할 수 있다.

예를들어 다음 방식은 사용자의 아이디와 비밀번호를 인증하는 방식의 strategy를 사용한다

위 예제에서 중요한 개념을 소개하는데, Strategies는 verify Callback 이라는 것을 요구한다. verify callback을 사용하는 목적은 credential을 가지고있는 사용자를 찾기 위해서이다.

Passport가 요청인증을 할때, request에 포함되어 있는 credential을 분석하는데, 이때 passport는 credentials를 인자에 넣고 verify callback 를 호출하게된다(위 예제일 경우 username과 password이다). credential이 유효할 경우 verify callback 은 done 을 호출해서 passport한테 사용자 인증이 성공되었다고 알려준다

return done(null, user); //사용자 인증 성공

만약에 credential이 유효하지 않을경우(예를들어 이메일이나 비밀번호가 일치하지 않을 경우) done 은 user가 아닌 false를 인자로 갖고 반환되어서 passport한테 사용자 인증이 실패 되었다고 말해줘야한다

return done(null,false,{message:”실패 메시지”}); // 인증 실패

마지막으로 예외가 발생할 경우(예를들어 데이터베이스 연결에 오류가 발생할 경우) done은 error를 인자를 갖고 반환되어야 한다.

return done(error); // 기타 오류(서버 오류)

미들웨어

Express의 미들웨어에 passport.initialize() 를 해서 passport를 초기화 해줘야 한다. 만약에 우리의 애플리케이션이 로그인 세션을 사용할 경우 passport.session() 미들웨어를 등록 해줘야 한다.

session을 사용하는것은 필수가 아닌다. 하지만 만약에 세션을 사용하게 될 경우 session() 을 등록 해줘야 한다

세션

웹 애플리케이션 에서, 사용자를 인증하기 위한 credential은 로그인 요청이 있을때만 전달이 된다. 사용자 인증이 성공하면 세션은 형성되고 쿠키에 저장된다.

각 요청에는 자격 증명이 아니라 세션을 식별하는 고유 한 쿠키가 포함됩니다. 로그인 세션을 지원하기 위해 Passport는 사용자 인스턴스를 세션과 직렬화 및 역 직렬화를 한다.

Username&Password 으로 로그인 인증

아이디 비밀번호로 사용자 인증을 하기 위해서 passport-local을 설치해서 사용하면 된다.

$ npm install passport-local

이후 passport-local 에 대한 설정을 해주면 된다

여기서 username과 password는 HTML form에 있는 input 태그의 name과 동일 해야한다.

라우팅

인증 성공 혹은 실패시 서로 다른 페이지로 라우팅 할 수 있다.

failureFlash:true 를 할경우 콜백에 있는 message 를 통해서 error 메시지를 전달할 수 있다.

LocalStrategy의 매개변수

LocalStrategy 는 기본적으로 username 과 password 를 찾으려고 한다. 만약에 username 과 password 를 다른 이름으로 사용하고 싶을 경우 다음같이 바꿔주면 된다.

username 와 password 를 email 와 passwd 로 바꾼것이다.
