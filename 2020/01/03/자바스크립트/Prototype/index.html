Prototype

How porototype works in JavaScript:

A Person constructor function is created during function declaration (Syntax analysis stage) and when the function is created, the prototype of the function is also created. Rather than storing isHuman inside the function, the isHuman property is stored inside Person.prototype and is shared by all its instances.
To create an instance of Person constructor function, the new keyword is used and 'Lee','Kim' are passed as arguments to the constructor function. Before using the new keyword Javascript engine always checks whether the function is a constructor or not.

First of all, when a function is created, Javascript implicitly implements so-called FunctionCreate abstract operation. This FunctionCreate checks whether the function has [[construct]] internal slot.

FunctionCreate(kind,ParameterList,Body,Scope[, prototype])
the first parameter kind can have one of the 3 arguments: Normal, Method, Arrow.
If a function’s kind is Normal (Functions created using Function declaration and Function Expression), the function is considered as a constructor function and can use new keyword to create instances.

By using the example above the function is created using function declaration. This means the function’s kind is Normal and is able to use new keyword to create instances.
When the new keyword is used with the constructor function:

  1. An empty object is created and the prototype of constructor function is bound to the instance, thus creating a link between this keyword and the instance itself.
  2. the instance is initialized inside the function. since the this keyword is bound to the instance, this.name = name will change to person1.name = name. the argument 'Lee' will be passed to the parameter and 'Lee' will be initialized.
  3. At the end of instance creation stage, the instance itself will be returned and will be stored in person1 variable.

Now, person1 and person2 are created and both variables are able to inherit properties and methods from Person.prototype. to access the properties from its parents, the instances can use __proto__ property to access its parent’s prototype. As stated above, person1 variable can use isHuman property even if the property is not stated inside in the instance itself.
