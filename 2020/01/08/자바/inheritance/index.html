자바 상속(extends)

객체지향은 특징으로서 새로운 클래스를 정의 할 때 이미 구현된 클래스를 상속(inheritance) 받아서 속성이나 기능이 확장되는 클래스를 구현할 수 있다.

상속은 기존에 존재하는 클래스의 기능을 좀더 확장해서 사용하고 싶을때 사용한다

사용방법:

자식 클래스 extends 부모클래스`: 자식 클래스가 부모클래스한테서 상속을 받는다는 뜻이다.

  • 다중 상속을 할 수 없다: 클래스C 가 클래스A, 클래스B한테서 상속을 받을 수 없다.(부모 클래스는 무조건 한명). 자바는 single inheritance 만 허용한다

상위 클래스는 하위 클래스보다 일반적인 개념과 기능을 갖는다

하위 클래스는 상위 클래스보다 구체적인 개념과 기능을 갖는다

생성자함수와 초기화 블럭(클래스 초기화 블럭/인스턴스 초기화 블럭)은 상속되지 않는다

자식 클래스의 멤버 개수는 조상 클래스보다 항상 같거나 많다

접근제어자가 Private 이거나 default일 멤버들은 상속되지 않는다기 보다는 상속은 받지만 자식 클래스가 접근을 할 수없는것이다.

두개의 클래스(자식 클래스 A 와 자식 클래스 B)가 한개의 부모 클래스 한테서 상속을 받더라도 자식 클래스 A 와 B는 아무런 관계가 없다. 상속 관계에서 형제 관계는 없다.

하위 클래스는 상위 클래스의 기능을 마음대로 사용할 수 있지만, private 한 변수와 메소드는 사용할 수 없다. 만약에 외부 클래스는 내 private 변수/메소드를 사용할 수 없지만 내 자식 클래스는 사용할 수 있게 하고 싶을 경우 private 이 아닌 protected 를 사용하면 된다.

private 는 선언한 클래스 내부에서만 사용할 수 있다. 상속을 한다고 해도 사용할 수 없다.

protected 를 사용해서 내 private 변수/메소드를 오직 내 자식 클래스들만 쓸수 있게 만들 수 있다.

예를들어 포유류 라는 부모클래스를 인간이 상속받는다: class Human extends Mammal{}

Object 클래스 - 모든 클래스의 조상

Object클래스는 모든 클래스 상속 계층도의 최상위에 있는 조상 클래스이다. 다른 클래스로부터 상속받지 않는 모든 클래스들은 자동적으로 Object 클래스로부터 상속받게 된다.

예를들어 아무한테서도 상속을 받지 않는 class Tv{}를 만들었을때, 컴파일러는 자동으로 class Tv extends Object{}으로 변환을 한다.

모든 클래스들과 그의 자식 클래스들이 Object 클래스로부터 상속을 받기 때문에 모든 클래스들은 Object 클래스의 멤버들을 사용할 수 있다. toString(), equals()같은 메서드들을 정의하지 않고 사용할 수 있는 이유는 해당 메소드들이 Object클래스에 정의되어 있기 때문이다.

접근 제어자 정리

외부 클래스 자식 클래스 동일 패키지 내부 클래스
public O O O O
protected X O O O
default X X O O
private X X X O

상속에서 클래스 생성 과정과 형 변환

하위 클래스가 생성되는 과정

하위 클래스가 생성될 때 상위 클래스가 먼저 생성된다(new 하위 생성자클래스()를 통해 하위클래스를 생성할려고 할때 상위 클래스의 생성자가 먼저 호출이된다.)

super는 자손 클래스에서 조상 클래스로부터 상속받은 멤버를 참조하는데 사용되는 참조변수이다. 멤버변수와 지역변수의 이름이 같을때 this를 사용해서 구별하듯이 상속받은 멤버와 자신의 클래스에 정의된 멤버의 이름이 같을 때 super를 사용해서 구별할 수 있다.

super()도 this()와 마찬가지로 역시 생성자이다. this() 는 같은 클래스의 다른 생성자를 호출할 때 사용하지만 super()는 조상 클래스의 생성자를 호출하는데 사용한다.

자손 클래스의 인스턴스를 생성하면, 자손 멤버와 조상 멤버가 모두 합쳐져진 하나의 인스턴스가 생성된다. 그래서 자손 클래스의 인스턴스가 조상 클래스의 멤버들을 사용할 수 있는 것이다. 이때 조상 클래스 멤버의 초기화가 수행되어야 하기 때문에 자손 클래스의 생성자에서 조상 클래스의 생성자가 호출 되어야 한다.

*생성자의 첫 줄에서 조상 클래스의 생성자를 호출해야 하는 이유는 자손 클래스의 멤버가 조상 클래스의 멤버를 사용할 수도 있으므로 조상의 멤버들이 먼저 초기화되어 있어야 한다. 이와 같은 조상 클래스 생성자의 호출은 클래스의 상속 관계를 올라가면서 계속 반복된다. 마지막으로 모든 클래스의 최상위 계층인 Object 클래스가 나오는데 Object클래스는 상위 클래스가 없기 때문에 Object 클래스 생성자의 내부에서 super()를 호출 하지 않는다. *

Object 클래스를 제외한 모든 클래스의 생성자는 첫 줄에 반드시 자신의 다른 생성자 또는 조상의 생성자를 호출해야 한다. 그렇지 않으면 컴파일러가 자동으로 첫번째 줄에 super()를 추가해준다.

사용자가 super() 를 지정해주지 않을 경우 컴파일러가 호출하는 super() 는 상위 클래스의 디폴트 생성자를 호출하는것과 동일하다. 부모 클래스의 특정 생성자를 호출하고 싶으면 꼭 super() 를 직접 호출 해줘야 한다. 한가지 주의해야할 것은 부모클래스가 디폴트 생성자를 가지고 있지 않을경우 컴파일 에러가 발생하고, 사용자가 부모 클래스에 지정되어 있는 생성자 형식으로 super()를 호출 해줘야 한다.

super()를 직접 호출하는 방법은 다음과 같다

상위 클래스의 생성자가 호출되고 하위 클래스의 생성자가 호출된다. 하위 클래스의 생성자 에서는 무조건 상위 클래스의 생성자가 호출되어야한다.

하위 클래스에서 상위 클래스의 생성자를 호출하는 코드가 없는 경우 컴파일러는 상위 클래스 기본 생성자를 호출하기 위한 super()를 추가한다.

this 키워드는 자기 자신의 메모리를 가리킨다. super()는 상위 클래스의 메모리 위치(참조값)을 가지고있다(super는 참조변수이다).

super()로 호출되는 생성자는 상위 클래스의 default constructor(혹은 constructor) 이다

만약 상위 클래스의 default constructor가 없는 경우(매개변수가 있는 생성자만 존재하는 경우) 하위 클래스는 명시적으로 상위 클래스의 생성자를 호출해야한다.부모 클래스의 디폴트 생성자에 인수를 전달해야할 경우 컴파일러가 super()를 호출할 수 없고, 개발자가 직접 super() 를 호출해야한다.

상위 클래스의 인스턴스가 먼저 생성이 되고 항위 클래스의 인스턴스가 생성된다.

상위 클래스로의 묵시적 형 변환(Upcasting)

상위 클래스 형으로 변수를 선언하고 하위 클래스 인스턴스를 생성할 수 있다

하위 클래스는 상위 클래스의 타입을 내포하고 있으므로 상위 클래스로 묵시적 형변환이 가능하다

상속 관계에서 모든 하위 클래스는 상위 클래스로 묵시적 형 변환이 된다

하위클래스는 상위 클래스로 묵시적 형변환이 가능하지만 상위클래스에서 하위클래스로 형변환을 할 수 없다

형 변환 에서의 메모리

상위 child = new 하위() 가 있을때, child 변수는 상위 클래스의 멤버 변수만 사용할 수 있다.

child인스턴스는 하위 클래스의 인스턴스이다. 즉 메모리 상에서는 하위 클래스 멤버변수와 상위 클래스 멤버변수가 다 만들어 져있지만, 사용 가능한거는 상위 클래스의 멤버 변수인것 이다.
