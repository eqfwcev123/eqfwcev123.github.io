Posted on 2020-01-28 In Django , Server Deploy , Network

Amazon Relational Database Service (RDS)

RDS를 사용하는 이유: AWS 클라우드 에서 관계형 데이터베이스를 설치, 운영, 확장할 수 있는 웹서비스이다. RDS를 EC2위에 설치하면 RDS는 EC2위에서 동작한다.

Virtual Private Cloud(VPC) 는 사용자의 AWS 계정 전용 가상 네트워크이다. VPC는 AWS 클라우드에서 다른 가상 네트워크와 논리적으로 분리되어있다. Amazon EC2인스턴스 같은 AWS 리소스를 VPC에서 실행할 수 있다.

RDS 데이터베이스를 생성할때 다음 항목들을 작성해줘야 한다.

  • DB 인스턴스 식별자 : 데이터베이스 인스턴스 이름
  • 마스터 사용자 이름 : 데이터베이스 username
  • 마스터 암호 : 데이터베이스 password

데이터 베이스 인스턴스를 생성하면 앤드포인트와 포트번호를 알 수 있다.

  • Endpoint: 아마존 RDS 데이터베이스 인스턴스와 연결 하기위해 필요한 데이터이다. 쉽게 말해 데이터베이스의 접근 경로이다.
  • 포트번호: 각 데이터베이스마다 디폴트 포트번호를 가지고 있다. 데이터베이스 가 listen 하고있는 포트 번호를 의미한다.

장고에서 Postgresql 설정하기

psycopg2를 사용하는 이유:

PostgreSQL데이터베이스는 여러개의 언어와 호환이 되는데, 그중 가장 자주 사용되는 언어가 파이썬이다. 파이썬을 이용해서 PostgreSQL과 소통을 할때 사용하는것이 psycopg2이다. psycopg2를 이용하면 파이썬의 데이터 타입을 postgresql의 데이터타입에 맞출 수 있다.

SQLite를 이용할때는 장고에서 알아서 SQLite로 변환을 시켜주지만 PostgreSQL을 사용할 때는 파이썬을 PostgreSQL로 바꿔주는 기능이 필요한데, 이것을 psycopg2 가 해준다.

pip install psycopg2 ==> psycopg2 설치

장고에서 ORM을 자동으로 SQL문으로 변환시켜주는 과정:

(변환 시작) Django ORM ==> SQL ==> Python ==> psycopg2 ==> PostgreSQL (변환 끝)

Amazon Simple Storage Service(S3)

*S3는 클라우드상에 스토리지를 제공하는 서비스이다. S3는 클라우드상에서 파일을 공유할 수 있는 서비스 이므로 인터넷에 접속할 수 있는 환경이라면 세계 어느 곳에서든지 데이터에 접속할 수 있다. *

S3 에서 자주 사용하는 용어

  • 버킷(Bucket): 데이터의 용기
  • 오브젝트(Object): 사용자가 버킷에 저장하는 파일을 호칭

*S3를 이용할 때 우선 데이터를 담는 그릇인 버킷을 작성한다. 그 버킷 속에 임의의 데이터를 저장할 수 있다. *

저장한 데이터에 접속할 때는 S3버킷을 지정하므로 버킷 이름은 모든 리전에서 유일해야 한다. S3상에 존재하는 버킷과 같은 이름을 사용할 수 없다.

버킷에 들어있는 데이터를 공개할 수 있다. 데이터를 모두한테 공개할 수 있고, 몇몇 한테 접근 권한을 설정하고 데이터를 공개할 수 있다.

장고에서 S3 사용하기

Django 에서 S3를 사용 하려면 boto3 와 boto3라이브러리 기반인 S3Boto3Storage 를 설치 해줘야 한다.

Boto3: boto3 는 AWS 의 SDK로 AWS의 다양한 서비스(S3, EC2 등등)를 사용할 수 있게 해준다.

pip install boto3 를 이용해서 boto3 설치

boto3를 사용하기 이전에 AWS에 대한 사용자 정보 인증을 해줘야 한다. 사용자 인증은 IAM을 생성 하고나서 AWS CLI 를 이용해서 인증을 하거나 직접 인증 을 해줘야 한다.

aws_access_key_id : IAM 아이디

aws_secret_access_password : IAM 시크릿 비밀번호

S3Boto3Storage: boto3 라이브러리 기반으로 아마존 S3를 사용할때 이용한다.

pip install django-storages 를 이용해서 django-storages(S3Boto3Storage 포함)

미디어/정적 파일을 S3에 올리기 위해서 다음 명령어를 사용한다 :

  • DEFAULT_FILE_STORAGE = 'storages.backends.s3boto3.S3Boto3Storage'
  • AWS_STORAGE_BUCKET_NAME = AWS Storage 버켓 이름
