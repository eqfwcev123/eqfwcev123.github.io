브로드캐스팅 주소 계산법

알아야할 용어들:

  • 네트워크 주소: 하나의 네트워크를 통칭하기 위해 사용하는 주소.
  • 브로드캐스 주소: 특정 네트워크에 속하는 모든 호스트/클라이언트 들이 듣게되는 주소를 의미한다. 네트워크 주소와는 반대로 특정 네트워크의 맨 마지막 주소를 브로드캐스르 주소로 사용한다.

IP주소는 네트워크 주소와 호스트주소로 나누어진다. IP주소를 네트워크 주소와 호스트 주소로 구분해 주는것이 서브넷 마스크(Subnet mask)이다. 추가적으로 서브넷은 추가적인 서브넷 서브 네트워크가 필요할 경우 IP주소 중 호스트 주소를 서브넷과 호스트주소(<네트워크주소><서브넷><호스트주소>) 로 나누어준다.

서브넷 마스크라고 불리는 이유는 서브넷 마스크를 이용해서 IP주소의 네트워크 주소를 구분하기 위해서 이다.

서브넷 마스크는 32비트 숫자로 구성된 IP주소이며, IP주소를 네트워크 주소와 호스트 주소로 나눈다. 서브넷 마스크는 32비트 숫자들 중 네트워크 주소를 모두 1로 표현하고 호스트 주소를 0으로 표기한다. 주어진 네트워크에서 두개의 호스느 주소는 특정 목적을 위해 예약되어있어서 호스트한테 할당될 수 없다. 0 인 주소는 네트워크 주소에 할당되어 있고, 255는 broadcast address이기 때문에 호스트에게 할당될 수 없다.

– AND 연산 –

1 AND 1 = 1

1 AND 0 = 0

0 AND 1 = 0

0 AND 0 = 0

1) 11000000 10101000 10111100 10000011 // IP 주소: 192.168.188.131

11111111 11111111 11111111 00000000 // 서브넷 마스크: 255.255.255.0

AND(&) 연산

11000000 10101000 10111100 00000000 // 네트워크 주소: 192.168.188.0

192.168.188.0 은 C 클래스 이기 때문에 브로드 캐스트 주소는 할당할수 있는 가장 큰 값 인

192.168.188.255 // 브로드캐스팅 주소

2) 11000000 10101000 10111100 10000011 // IP 주소: 192.168.188.131

11111111 11111111 11111111 10000000 // 서브넷 마스크: 255.255.255.128

AND(&) 연산

11000000 10101000 10111100 10000000 // 네트워크 주소: 192.168.188.128

192.168.188.255 // 브로드캐스팅 주소

3) 11000000 10101000 00010000 00000001 // IP 주소: 192.168.16.1

11111111 11111111 11111111 11100000 // 서브넷 마스크: 255.255.255.224

AND 연산

11000000 10101000 00010000 00000000 // 네트워크주소: 192.168.16.0

192.168.16.31 // 브로드캐스팅 주소

4) 11000000 10101000 10111100 01111000 //IP주소: 192.168.188.120

11111111 11111111 11111111 10000000 // 서브넷 마스크: 255.255.255.128

AND(&) 연산

11000000 10101000 10111100 00000000 // 네트워크 주소 192.168.188.0

192.168.188.127 // 브로드캐스팅 주소

서브넷 마스크 값이 최대값이 128이면 2개로 네트워크를 나누는 것인데(0127, 128255) 이중 ip가 어느 서브넷에 속하는지 판단하고 그 주소에서 표현 가능한 최대로 적은 값이 네트워크 주소이고 가장 큰 값이 브로드캐스트 주소값이다.

구분 방법

서브넷 이

x.x.x.0 이면 0~255까지 모든 IP를 하나로 간주 하겠다.

x.x.x.128 이면 0 ~ 127, 127 ~ 255로 2개로 분리한다

x.x.x.192 이면 0 ~ 63, 64 ~ 127, 128 ~ 191, 191 ~ 255로 4개로 분리한다

x.x.x.224 이면 0 ~ 31, 32 ~ 63, 64 ~ 95, 96 ~ 127, 128 ~ 159, 160 ~ 191, 192 ~ 223, 224 ~ 255로 총 8개로 분리한다

브로드캐스팅을 더쉽게 구하는 방법

11111111 11111111 11111111 11100000 // 서브넷 마스크: 255.255.255.224

00000000 00000000 00000000 00011111 // 서브넷 마스크 invert

11000000 10101000 00010000 00000001 // IP 주소: 192.168.16.1

00000000 00000000 00000000 00011111 // 서브넷 마스크 invert

Logical OR 하기

11000000 101010000 00010000 00011111 // 브로드캐스팅 주소 192.168.16.31

위에서 사용한 예제)

11000000 10101000 10111100 01111000 //IP주소: 192.168.188.120

11111111 11111111 11111111 10000000 // 서브넷 마스크: 255.255.255.128

00000000 00000000 00000000 01111111 // 서브넷 마스크 invert

IP 주소와 서브넷 invert Logical OR

11000000 10101000 10111100 01111111 // 브로드캐스팅 주소: 192.168.188.127
