인스턴스 변수

클래스의 영역에 선언되며, 클래스의 인스턴스를 생성할 때 만들어진다. 그렇기 때문에 인스턴스 변수의 값을 읽어 오거나 저장하기 위해서는 먼저 인스턴스를 생성해야한다. 인스턴스는 독립적인 저장 공간 을 가지므로 서로 다른 값을 가질 수 있다. 인스턴스마다 고유한 상태를 유지해야하는 속성의 경우 인스턴스 변수를 선언 한다

클래스 변수(static)

클래스 변수를 선언하는 방법은 인스턴스 변수 앞에 static을 붙이기만 하면 된다. 인스턴스마다 독립적인 저장공간을 갖는 인스턴스 변수와는 달리, 클래스 변수는 모든 인스턴스가 공통된 저장공간을 공유하게 된다. 한 클래스의 모든 인스턴스들이 공통적인 값을 유지해야 하는 속성의 경우, 클래스 변수를 선언해야한다.

클래스 변수는 인스턴스변수와 달리 인스턴스를 생성하지 않고도 사용할 수 있다. 하지만 인스턴스를 생성하지 않고 사용을 하기 때문에 클래스명.클래스변수 형식으로 사용한다. People 이라는 클래스가 있고, isHuman 이라는 클래스 변수가 있을 경우 People.isHuman 을 해서 클래스 변수를 호출 한다.

클래스가 메모리에 로딩될 때 생성되어 프로그램이 종료될 때 가지 유지되며, public 을 앞에 붙이면 같은 프로그램 내에서 어디서나 접근할 수 있는 전역 변수의 성격을 갖게된다.

지역 변수

메서드 내에 선언되어 메서드 내에서만 사용할 수 있는 변수이다. 메서드가 종료되면 소멸되어 사용할 수 없게 된다. for문 이나 if 문, 또는 while 문의 블록 내에 선언된 ({}로 감싸져 있는 함수혹은 문)변수들은 지역변수이며 해당 문이 종료될 때 내부에서 선언된 인스턴스 변수도 소멸한다.

인스턴스 변수는 인스턴스가 생성될 때 마다 생성되므로 인스턴스마다 각기 다른 값을 유지할 수 있지만, 클래스 변수는 모든 인스턴스가 하나의 저장 공간을 공유하므로 항상 공통된 값을 갖는다.

멤버변수와 지역 변수의 메모리 구조

자바에서 메모리를 3등분을 시키는데 만약 우리가 프로그램을 컴파일 시키면 먼저 모든 클래스 파일을 메모리의 최상단 부분에 로딩 시킨다. 이때 컴파일된 클래스 파일이 로드될 때 클래스 변수들도 모두 할당이 된다. 즉 인스턴스 생성 없이 바로 클래스 변수(static 변수)가 생성이된다. 클래스 파일의 정적 메소드와 정적 변수를 저장하는 공간을 메소드 공간(Method Area)라고 부른다

Call stack 영역에는 메인 메소드와 지역 변수, 그리고 메소드의 매개변수가 저장이된다. Call Stack 에 들어가는 데이터는 동적 데이터로서 프로그램 실행도중 함수가 호출이되면 해당 함수가 콜스텍에 쌓이게 된다. 함수가 호출되면 해당 함수가 갖고있는 모든 지역 변수들이 콜스텍에 생성이 되고, 함수가 종료될때 내부의 지역변수들도 삭제된다. 스택 영역에는 임시로 생성되는 데이터들이 저장이 된다. 함수처럼 함수호출이 실행되면 콜스텍에 생성이 되고, 함수호출이 종료되면 사라지는 형식이다.

인스턴스를 생성하게 되면 참조변수가 힙영역 안에있는 해당 객체(인스턴스)를 가리킨다. 메모리상 최 하단의 마지막에 Heap 인스턴스 객체가 생성이 된다. 다시말해 인스턴스 변수도 힙 영역에 들어가게 된다. 인스턴스 변수는 콜스텍에 있는 지역 변수와 다르게 함수호출이 끝나도 사라지지 않는다. 인스턴스 변수가 사라지는 시점은 인스턴스변수를 아무도 참조를 하지 않아서 가비지 컬렉터(Garbage collector)가 가져갈때이다(혹은 JVM이 종료될때).

자바의 8가지 원시타입(Primitive type)을 제외한 다른 타입들은 레퍼런스 변수(참조 변수)이다.

참조변수 들은 실행될 때마다 많은 데이터들을 스택 메모리 영역에 뒀다 뺐다 하는게 비효율 적이므로, 힙 영역에 그 내용이 저장되고, 스택 메모리에는 주소만 저장을 한다.

기본형 매개변수와 참조형 매개변수

자바에서는 메서드를 호출할 때 매개변수로 지정한 값을 메서드의 매개변수에 복사해서 넘겨준다. 매개변수의 타입이 기본형일 때는 기본형 값이 복사되지만, 참조형이면 인스턴스의 주소가 복사된다.

메서드의 매개변수를 기본형으로 선언하면 단순히 지정된 값만 얻어오지만, 참조형으로 선언하면 값이 저장된 곳의 주소를 알 수 있기 때문에 값을 읽어 오는 것은 물론 값을 변경하는것도 가능하다.

기본형은 값을 복사, 참조형은 값의 주소값을 복사

참조형 반환타입

매개변수 뿐만 아니라 반환타입도 참조형이 될 수 있다. 반환 타입이 참조형이라는 것은 반환하는 값의 타입이 참조형이라는 건데, 모든 참조형 타입의 값은 객체의 주소값 이므로 그저 정수값이 반환되는 것일 뿐 특별한 것은 없다.

d 라는 인스턴스를 생성하고 인스턴변수 x 에 10을 할당했다. 새로운 Data타입 참조변수 d2를 생성하고 copy(d)의 반환값을 할당하는데, copy 함수에 d인스턴스를 인자로 넣어주었다. copy 함수는 d 인스턴스를 받고, 내부에 새로운 참조변수인 tmp를 생성하고 tmp인스턴스를 가리키게 만든다. tmp.x = d.x 를 하면서 tmp참조변수가 가리키는 인스턴스 객체의 x 인스턴스 변수는 d 참조변수가 가리키는 인스턴스 객체의 x 인스턴스 변수와 동일하게 만든다. 아래 그림을 보면

d 는 0x100이라는 주소값을 갖고 tmp는 0x101이라는 주소값을 갖는다. tmp.x = d.x를 하면 0x101.x = 0x100.x 를 하는것과 동일하며 tmp.x 는 d.x와 동일하게된다. copy함수가 종료되며 tmp의 주소값인 0x101이 main함수에 반환되고 d2변수는 tmp의 주소값인 0x101을 갖는다. 마지막에 println()을 해보면 d1.x 와 d2.x의 값이 동일하다는걸 알 수 있다.
