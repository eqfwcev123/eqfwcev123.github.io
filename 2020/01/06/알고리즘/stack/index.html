Stack 자료구조

Posted on 2020-01-06 In Algorithm

Stack 자료구조

  • 데이터를 제한적으로 접근할 수 있는 자료구조
    • 한쪽 끝에서만 자료를 넣거나 뺄 수 있는 구조
  • 가장 나중에 쌓은 데이터를 가장 먼저 빼낼 수 있는 데이터 구조

스텍 구조

  • 스택은 LIFO(Last in First Out) 데이터 관리 방식을 따른다
  • 대표적인 스택의 활용 내용
    • 컴퓨터 내부의 프로레스 구조의 함수 동작 방식
  • 주요기능
    • push(): 데이터를 스택에 넣기
    • pop(): 데이터를 스택에서 꺼내기

스택 구조와 프로세스 스택

  • 스택 구조는 프로세스 실행의 가장 기본 구조이다

![스택 예제](/Users/idohyeon/Desktop/스택 예제.png)

자료구조 스택의 장단점

  • 장점

    • 구조가 단순해서 구현이 쉽다
    • 데이터 자장/읽기 속도가 빠르다

  • 단점(일반적인 스택 구현시)

    • 데이터 최대 개수를 미리 정해야한다
    • 저장 공간의 낭비가 발생할 수 있다

    스택은 단순하고 빠른 성능을 위해 사용되므로, 보통 배열 구조를 구현하는 것인 일반적이다. 이 경우, 위에서 열거한 단점이 있을 수 있다.
