IP주소란?

네트워크 환경에서 컴퓨터간 통시하기 위해 각 컴퓨터에 부여되는 네트워크 상의 주소이다.

맥 zsh에 ifconifg 라고 입력 하면 내 컴퓨터에 부여된 네트워크상의 주소가 나온다.

  • IPv4주소는 내 컴퓨터의 주소이다.
  • 192.168.219.144 가 있으면 192.168.219 는 네트워크 주소이고 144는 호스트 주소이다(내 컴퓨터에 부여된 호스트 IP 이다. 내가 서울특별시 강남구 논현동 134-3 201호에 살면 서울특별시 강남구 논현동 134-3이 네트워크 주소이고, 201호가 호스트IP가 된다.)
  • IPv4의 마지막 자리가 무조건 호스트 주소는 아니다. IP주소의 클래스에 따라 달라진다.
  • 공통적인 공유되는 주소가 네트워크 주소, 내 자신을 식별해주는 마지막 자리가 호스트주소 이다.

IPv4 는 IP주소를 32자리로 표현한 것이다

192.168.24.133 이라는 주소가 있을때, 이 주소는 2진수 를 10진수로 표기한 것이다.

10진수를 2진수로 표기하게 되면 0~2^32, 약 43억개의 IP 주소를 사용할 수 있는것이다. 이전에는 충분할거라고 생각했지만 요즘은 휴대용기기를 포함해 여러개의 기기를 가질수 있기 때문에 IP주소가 부족해졌다. 따라서 현재는 IPv4에서 IPv6표현식으로 대체하는 중이다

IP주소 A,B,C 클래스

IP주소 클래스: 192.168.24.133 이라는 주소가 있을경우 마지막 133은 호스트 주소이고 앞에 세 숫자는 네트워크 주소이다. 호스트주소와 네트워크 주소를 구분할 수 있는 것은 클래스 때문이다.

IP 주소에는클래스라는 개념이 있고, 클래스를 통해서 어디까지가 네트워크 주소이고 어디까지가 호스트 주소인지 알 수 있다. 다시말해 클래스는 하나의 IP주소에서 네트워크와 호스트 영역을 나누는 방법이자 약속이다.

IP주소를 A,B,C 클래스로 나누는 이유는 네트워크 크기에 따른 구분이라고 생각하면 된다. 하나의 네트워크에서 몇개의 호스트 IP까지 가질 수 있는지에 따라 클래스를 나눌수 있다.

A 클래스

A 클래스는 하나의 네트워크가 가질 수 있는 호스트 수가 제일 많은 클래스이다. IP주소를 32자리 수로 표현했을 때 맨 앞자리가 0인 경우가 A 클래스이다.

0xxx xxxx.xxxx xxxx.xxxx xxxx.xxxx xxxx 는 A 클래스이다.

A 클래스의 범위는 0000 0000. 0000 0000. 0000 0000. 0000 0000 ~ 0111 1111. 1111 1111.1111.1111.1111(0.0.0.0 ~ 127.255.255.255) 까지이다

A 클래스는 첫번째 옥테드(첫번째 0)은 네트워크 주소를 의미하고 나머지 3개의 옥테드는 호스트 주소이다.

A클래스에서 네트워크 주소는 가장 작은 네트워크인 1.0.0.0과 가장 큰 네트워크는 126.0.0.0까지로 규정 되어있다.(127은 제외되는데 이는 일종의 약속이다.)

B 클래스

B 클래스는 반드시 10(2진수표기법)으로 시작한다.

10xx xxxx. xxxx xxxx. xxxx xxxx. xxxx xxxx

128.0.0.0 ~ 191.255.255.255

첫 2옥테드가 네트워크 주소, 끝에 두 옥테드가 호스트주소

C 클래스

C클래스는 반드시 110(2진수표기법)으로 시작한다

110x xxxx. xxxx xxxx. xxxx xxxx. xxxx xxxx

192.0.0.0 ~ 223.255.255.255

첫 3옥테드가 네트워크 주소, 마지막 옥테드가 호스트 주소

요약
