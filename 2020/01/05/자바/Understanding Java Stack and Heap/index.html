Understanding Process, Thread and Data structures

Posted on 2020-01-05 Edited on 2020-01-06 In Java

Process

  • A program in execution is often referred as process and a thread is a subset of a process.

  • A process can either contain one thread (Single thread process) or mutiple threads(Multi-thread process).

  • Inside a Process contains 4 different memory area: Data, Code, Stack, Heap

Thread

A unit of work that runs independently within a process. Specifically, a unit of work or task managed by the operating system. Threads and tasks (or tasks) can be used interchangeably.

  1. A single process is occured by JVM and statements within main()are a single thread.
  2. To create another thread other than main (), either inherit the Thread class or implement the Runnable interface.
  3. In multi-threaded operation, each thread can send and receive information to reduce errors in processing.
  4. Processes cannot send or receive information with each other.

Whenever a process is executed, it is allocated with 4 different memory area: Code, Data, Stack and Heap. Bascially, a thread is simulataneously created when a process is created. This thread is called the main thread and unless a new thread is created, every program codes will be executed on the main thread. Every process posses its own memory address and will be executed on its own memory address. In order for a thread to access another thread’s resources(variables or methods), a inter-process communication (IPC) is required between each threads.

Code Area

The Code Area, also called the Text area, is an area where codes that will be executed by the program are stored. codes inside the Code Area are accessed by CPU and will be compiled one at at time.

Data Area

An area where Static Methods and Static variables are stored. Static variables/methods will be allocated at the beginning of the program and will be destroyed when the program ends.

Stack Area

The stack area of memory is the area where local variables and parameters related to function calls are stored. The stack area is allocated with the call to the function and is destroyed when the function call completes. The stack area stores data in push operations and fetches data in pop operations. This stack operates according to Last-In First-Out (LIFO), thus the latest stored data is fetched first.

Heap Area

The Heap area is an area which can be managed by the user. The heap area is dynamically allocated and freed by the user.

Stack(static memory allocation)

  • A stack is a type of data structure that stores data of the same type in a sequential one to one order. When we extract and insert data from the stack, we can only access the end of the sequence which is the data that is pushed most recently. The new data is placed on top of the stack, and when extracting data, we only have access to the most recently inserted data (top level).
    For this reason, the stack is often referred to as “LIFO data structure (Last In First Out)”, which means that the most recent entry comes first.
  • The stack is also used by the JVM to run Java programs. Each thread has one stack and tracks all the methods. Every time when new methods are called, a new frame is inserted on the stack, and each time the method ends, it is removed from the stack.
  • Data Types that are stored in stack area:
    • Integer type: byte/char/short/int
    • Real Integer type: float/double
    • boolean type: boolean
  • all the types state above are primititive types and have a fixed data size.

Heap(Heap memory allocation)

  • Dynamic memory allocation refers to the allocation of memory space for use during runtime in computer programming. When you’re done, you can return it for use by the operating system and reassign it the next time it’s requested. In contrary to static memory allocation, where the allocation of memory is made taking into account the amount of memory the program will use at the moment it runs.

  • Dynamically allocated memory space remains until the programmer explicitly releases it or the garbage collector(GC) releases the data. Dynamic allocation is allocated to the heap of the process. When the process terminates, the memory resources are returned to the operating system and is automatically freed. However, if the process continues to run, dynamically allocated space is retained, so if a program requests memory allocation beyond the size of a given heap, the memory cannot be allocated due to the lack of memory space. Therefore, it is more efficient for a programmer to use functions(methods) to return used heap area and free up heap area spaces. In Java, the Garbage collector automatically collects unused memory and free up the space.

  • Data Types that are stored in Heap area:

    • Array
    • Enum
    • Class
    • Interface

  • The heap area is a space where objects and arrays are created and stores reference types (arrays, enumerations, classes, interfaces) in the form of memory addresses.

  • The size of each data structures are flexible(size of the data are not fixed).

  • If the memory address is not referenced, the memory will be automatically freed by Garbage collector
