도커에 대한 간단한 설명

Posted on 2020-01-29 In Docker

웹서버

웹서버의 정의: 웹서버는 HTTP혹은 기타 프로토콜을 이용해 사용자의 응답에 맞춰서 결과를 응답해준다.

웹서버는 두가지 의미중 하나를 의미한다.

  1. 웹서버(소프트웨어): 웹 사용자가 어떻게 호스트파일들에 접근하는지를 관리한다.
  2. 웹 서버(하드웨어): 정적 파일을 저장하고 사용자의 요청에 맞게 정적파일을 돌려주는 기능을 가지고 있는 컴퓨터이다.

정적 웹서버는 request 가 들어오면 항상 같은 파일들을 돌려준다(FileSystem 으로 이동).

동적 웹서버는 request에 따라 다른 응답을 돌려준다(WSGI,Gunicorn, uWSGI).

웹서버의 종류: Apache, IIS, Nginx, LiteSpeed

WSGI, Gunicorn, uWSGI

웹서버에서 사용자의 요청을 보내면 정적/동적 응답을 해줘야 하는데, 정적 응답일 경우 FileSystem에서 파일을 찾아서 돌려준다. 동적 응답일 경우 WSGI, Gunicorn, uWSGI 을 이용해서 백앤드 파이썬 웹 어플리케이션으로 forwarding 해준다. 구체적으로 설명 하자면 webserver(Apache, Nginx)같은 경우 주로 C 언어로 작성이 되어있다(빨라야 하기 때문에). 사용자가 요청을 보낼경우 웹서버는 요청을 받아서 web application 으로 전달을 해야하는데 웹서버는 요청을 C 언어로 보내게된다. WSGI 나 Gunicorn 이 없을 경우 웹 애플리케이션에서 일일히 이걸 번역을 해야한다. WSGI는 웹 서버 혹은 웹 애플리케이션 에서 보내는 요청/응답을 해당언어(WSGI 같은 경우 파이썬)에 맞게 번역을 해주는것이다. 다시말해 WSGI는 웹서버와 장고 어플리케이션 간의 번역을 해주는 것이다(C에서 파이썬, 파이썬에서C).

python manage.py runserver 를 할 경우 runserver 는 웹서버, WSGI 와 디버거의 역할을 다 해주는 것이다.

Docker

도커는 컨테이너 기반의 오픈소스 가상화 플랫폼이다.

도커를 사용할때 알아야 할 4개의 키워드:

  • 컨테이너(Container): 컨테이너는 다른 공간에서 프로세스가 동작하는 기술이다. 기존의 가상화 기술과는 다른점을 가지고있다. 이전에 가상화 기술이라고 하면 virutal box 혹은 VMware 등의 가상 머신을 자기자신의 Host 컴퓨터의 OS (Host OS) 에 설치해서 사용하는 방식이였다. 매우 간단한 방법이지만 사용자의 OS 에 새로운 가상의 Guest OS 를 올리는것은 엄청난 overhead 를 발생 시키기 때문에 속도면에서 문제가 많았다. 이것을 해결하기 위해 나온것이 Docker 엔진이다. Docker 엔진 위에다가 virtual box/VMware에서 사용할려고 했던 기능들을 컨테이너에 담아서 올린다고 생각하면 된다.기존의 방식과 다르게 프로세스를 격리해서 사용하는 방식이다. Docker 를 사용하게되면 Host OS 위에다 Guest OS를 더이상 올리지 않아도 되게된다.
  • 이미지(Image): 컨테이너를 생성하기 위해 사용된다. 이미지를 일종의 CD라고 생각하면 된다. 예를들어 스타크래프트 CD를 이용하면 우리는 스타크래프트를 여러번 설치할 수 있다. 하지만 우리가 설치할 수 있는 것은 동일한 버전의 스타크래프트이다(최신 버전을 CD 로 설치할 수 없다). 이것을 Immtuable 하다고 한다. 이미지를 이용해서 여러개의 컨테이너를 생성할 수 있다. 이미지를 이용해 생성한 컨테이너에는 이미지에 저장되어 있는 설정만 존재하고, 추가적인 설정들은 컨테이너에서 추가한다. 이미지는 하드디스크에 저장된다.
  • Host OS : 내 컴퓨터의 Operating System
  • Guest OS : 가상 컴퓨터의 Operating System.

Docker 설치

  • brew cask install docker : Docker 설치

  • docker ps: 컨테이너들을 모두 표시한다.

  • docker login : docker로 로그인

  • docker images : 각 이미지의 크기

  • docker rmi [이미지이름] : 이미지 지우기

    • 은 우리가 버린 레이어이다. docker system prune 으로 삭제 할 수 있다.

  • docker run [이미지이름] : 새로운 컨테이너 에서 이미지의 커멘드를 실행한다.

    • docker run ubuntu 는 ubuntu 의 CMD 를 실행한다 ("/bin/bash")

docker run --rm -it ubuntu /bin/bash : 다른 우분투 운영체제로 진입

  • --rm 을 사용하면 컨테이너를 나갈때 컨테이너를 삭제한다. --rm 을 사용하지 않으면 컨테이너를 나가도 커테이너가 삭제되지 않는다. 컨테이너는 주로 데이터를 남겨둘때 사용하는게 아니다. 그렇기 때문에 데이터를 남겨두고 싶을경우 이미지에다 남겨야 한다.
  • --it 를 사용하면 사용자가 터미널에 추가적으로 입력을 할 수 있게 해준다.

docker run --rm -it python:3.7-slim /bin/bash : 파이썬 이미지 로 컨테이너를 사용. 이때 파이썬은 데비안 위에 올라가 있다. slim 은 이미지를 설치하는데 있어 최소한의 파일들만 설치하는 것이다.

장고 설치후 python manage.py runserver 0:8000 를 해도 localhost:8000 로 연결할 수 없다. 그 이유는 container는 외부와 연결이 되있지 않다.

  • Docker 의 컨테이너 네트워킹: 우리가 컨테이너를 생성할 때, 우리가 생성한 컨테이너는 외부로 포트를 열어 주지 않는다. Docker 외부에 있는 서비스로 포트를 열고 싶거나 그 반대로 하고 싶을 경우 --publish 혹은 -p flag를 사용해야한다. -p 를 사용하면 컨테이너 포트를 Docker host의 포트와 연결시킬 수 있다.

  • -p를 사용해서 컨테이너의 포트를 노출시킬 수 있다. 노출된 포트는 호스트가 접근 할 수 있고 호스트를 접근할 수 있는 유저들도 포트에 접근할 수 있다.

  • -p Docker호스트포트:컨테이너포트

  • -p 8080:80: 컨테이너의 TCP 포트 80번을 Docker 호스트의 포트 8080과 연결 시킨다.

docker build [option] PATH | URL |: Docker 파일을 이용해서 이미지를 생성한다.

  • -t: 이름설정. 태그 설정
  • -f: Docker 파일 이름

docker build -t instagram -f Dockerfile . :

  • -t 를 이용해서 instagram을 이미지 이름으로 지정
  • -f 를 이용해서 Dockerfile 사용
  • .은 Docker파일 위치

docker run --rm -it -p 8001:8000 instagram 으로 생성한 인스타그램 이미지 실행.

Docker 파일에서 python manage.py runserver 0:8000 을 했기 때문에 호스트의 8001번 포트로 접근하면 container 의 8000번 포트로 연결된다.

docker container 를 docker hub에 올리는 방법

docker 이미지를 docker hub에 올리기 위해서는 우선 로컬 이미지에 이름을 지어줘야한다(이미지의 태그가 해당 이미지의 이름이다).

이미지에 이름을 지어주는 방법은 다음과 같다:

  • 빌드할때 이름 부여: docker build -t <hub-user>/<repo-name>[:<tag>]
  • 이미 존재하는 이미지의 이름을 바꾸기 docker tag <existing-image> <hub-user>/<repo-name>[:<tag>]
  • *commit 할때: * docker commit <existing-container> <hub-user>/<repo-name>[:<tag>]

이름을 지어주고 나서 docker hub 에 push 를 하는 방법은 다음과 같다.

docker push [options] Name[:Tag]

instagram, ubuntu 는 공식 이미지이기 때문에 이름을 붙여줄 필요가 없다. 하지만 instagram은 공식이 아니기 때문에

docker tag instagram instagram 레포명 을 해줘야 한다
