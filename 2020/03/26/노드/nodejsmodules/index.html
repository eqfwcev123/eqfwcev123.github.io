Node.js 는 무엇인가?

Node.js –> 자바스크립트 런타임: 컴퓨터가 실행되고있는 동안 프로세스나 프로그램을 위한 소프트웨어 서비스를 제공하는 가상 머신이다. 다시말해 Node를 가상머신이고 프로그램을 위한 소프트웨어 서비스를 제공한다. Node가 설치되어 있으면 어떤 환경에서든 자바스크립트를 사용할 수 있게된다.

자바스크립트는 원래 브라우저에서만 동작을 했는데, 노드의 등장으로 인해 웹브라우저 밖(가상 머신) 에서 동작할 수 있게 됬다.

이벤트 루프

이벤트 루프. 자바스크립트 코드가 실행될 때 코드가 어떤 순서로 실행되는지. (스택과 테스크큐)

Task Queue에 들어가는 함수들: setTimeout, setInterval, setImmediate, Promise, async…await, 이벤트 리스터의 콜백 함수

Node의 핵심적인 개념

  1. 이벤트기반
  2. 논 블로킹 I/O: FileSystem, Network(request, response) 는 NonBlock으로 동작한다.
  3. 싱글 스레드: 한번에 한가지 일만 할수있다. 자바스크립트는 한번에 한가지 일만 할수있다.

Global 객체

웹개발을 할때 window 는 전역 객체를 의미한다. 하지만 노드에는 window 라는게 없고 global 이라는 객체를 사용한다.

전역객체는 어디서든 접근할 수 있다.

Console 객체

console.log()

console.error()

console.dir()

console.time()

console.timeEnd() : 전체 수행 시간

console.trace() : 함수 호출 추적

타이머 함수

setTimeout() : 몇초후 실행

setInterval() : 몇초마다 실행

clearTimeout() : setTimeout 해제

clearInterval() : setInterval 해제

setImmediate(): 즉시실행. 함수를 이벤트 루프로 보낼때 사용

clearImmediate() : 즉시실행 해제

__filename, __dirname

__filename: 현재 파일까지의 모든 경로

__dirname: 현재 파일을 포함하고 있는 상위 폴더 까지의 경로

Path 모듈 주요 함수들

path.basename(경로)

path.dirname(경로)

path.extname(경로): 파일 확장자 이름 (예: .js, .py)

path.basename(경로): 현재 파일의 확장자 까지 포함한 이름 (예: app.js, test.py)

path.join(경로, 추가경로) : 특정 경로뒤에 새로운 경로 추가. 절대 경로를 무시하고 합친다

path.parse(경로): 지정한 경로를 root, dir, base, ext, name 의 정보를 포함한 객체 형식으로 반환

path.format({root: , dir: , base: , ext: , name: }): root, dir, base, ext, name의 정보를 입력하면 해당 경로에 맞는 경로를 반환. path.parse() 의 반대라고 생각하면 됀다.

path.normalize(경로): 경로를 옳바르게 변경을 해준다.

path.isAbsolute(경로): 해당 경로가 상대 절대경로인지 상대경로인지 알려준다

./ 현재 폴더 상대 경로

../ 부모 폴더 상대 경로

/ 루트 절대 경로

path.relative(경로1, 경로2): 경로1이 경로2까지 가기위한 상대경로를 반환

url 모듈

url.format(url주소): 각각의 url 구성요소를 하나로 합쳐서 반환한다.

url.format(myURL)
// ‘https://user:pass@sub.example.com:8080/p/a/t/h?query=string#hash'

url.parse()

url.searParams

예시 url: http://www.gilbut.co.kr?page=3&limit=10&category=nodejs&category=javascript

  • url.searParams.getAll() : 쿼리에서 특정 key가 가지고있는 모든 값을 반환
    • url.searParams.getAll('category') 는 ['nodejs', 'javascript'] 를 반환
  • url.searParams.get(): 특정키의 값을 반환
    • url.searParams.get('limit') 은 10을 반환
  • url.searParams.has() : 특정 key를 가지고있는지 확인. Boolean 반환
    • url.searParams.has('page') 는 True를 반환
  • url.searParams.keys(): 쿼리스트링에 있는 모든 키값 반환
    • url.searParams.keys() 는 ['page','limit','category'] 반환
  • url.searParams.values() : 쿼리스트링에 있는 모든 value값 반환
    • url.searParams.values() 는 ['3','10','nodejs','javascript'] 반환
  • url.searParams.append(key, value) : 값을 추가한다
  • url.searParams.set(key,value) : 기존값을 초기화 후 수정한다
    • url.searParams.set('filter','es3') 을 할경우 filter=es3을 추가. url.searParams.set('filter','es6') 를 또할경우 filter=es6이 된다.
  • url.searParams.delete(key): 해당 key값을 갖는 쿼리스트링을 삭제
  • url.searParams.toString() : 키-값을 문자열로 반환한다.
