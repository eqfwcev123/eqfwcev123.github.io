Servlet 매핑

Servlet을 외부에서 요청하기 쉽도록 특정 문자를 이용해서 맵핑하는 방법

Servlet 을 Web Container 내부에 넣어놓고(tomcat), 사용자의 요청이 들어오면 web container 내부에있는 servlet이 응답을 해준다.

만약에 servlet이 엄청 많을경우, 브라우저가 정확히 어떤 servlet에 요청을 보내는지 알려줘야 한다. 그렇기 때문에 servlet마다 고유한 이름을 지정해준다.

기존: http://localhost:8090/[context-path]/servlet/com.package.servlet

Servlet 맵핑: http://localhost:8090/[context-path]/[고유한 이름]

이것을 Servlet 맵핑이라고 한다. 복잡한 이름을 사용하는대신 url이름을 간략하게 설정

Servlet 맵핑을 사용하기 위해서는 web.xml 파일을 이용하거나 Java Annotation을 사용하면 됀다. (주로 Java Annotation을 더 자주 사용한다).

web.xml 파일을 이용한 맵핑

Java Annotation 을 이용한 맵핑

Servlet 클래스 위에 @WebServlet("/se1")를 추가하면 web.xml에 따로 url-mapping을 해줄 필요 없이 url-mapping이 동작한다
