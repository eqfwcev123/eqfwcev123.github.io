GROUP BY, HAVING

GROUP BY 는 여러개의 동일 한 데이터를 가지는 특정 컬럼을합쳐주는 역할을 하는 명령어이다.

SQL 그룹 함수들: COUNT, MAX, MIN, SUM, AVG, VAR_SAMP, STDDEV

  • COUNT: 데이터를 그룹으로 묶고, 각 그룹마다 데이터를 몇개 가지고 있는지 확인
  • MAX: 최대값
  • DISTINCT: 중복되는 데이터를 제거한다
  • MIN: 최소값
  • SUM: 총합을 돌려준다
  • AVG: 평균값을 반환한다

HAVING

GROUP BY에서 반환되는 결과에 조건을 줄수 있다.

WITH ROLLUP

WITH ROLLUP을 사용할 경우 최종 결과에 추가적으로 총합을 넣어준다.

CREATE USE ALTER DROP

CREATE

데이터베이스 생성 혹은 테이블 생성

CREATE DATABASE <데이터베이스 이름>

CREATE TABLE <테이블 이름>

CREATE TABLE <테이블 이름>(column_name_1 column_data_type_1, column_constraint_1, column_name_2 column_data_type_2, column_constraint_2, ...)

USE

데이터베이스 사용

USE <데이터베이스 이름>

ALTER (ADD, MODIFY, DROP)

데이터베이스 혹은 테이블의 설정을 변경

ALTER DATABASE <데이터베이스 이름> <변경할 내용>

ex: ALTER DATABASE world CHARACTER SET=ascii

ex:ALTER DATABASE world CHARACTER SET = utf8

ALTER 를 이용해서 테이블의 컬럼을 추가하거나 삭제하거나 수정할 수 있다.

ALTER TABLE <테이블 이름> <변경방식> <변경할 내용>

ex: ALTER TABLE user2 ADD tmp TEXT –> 새로운 column을 추가한다

ex: ALTER TABLE user2 MODIFIY COLUMN tmp INT –>tmp column이 입력받을 타입 변경

ex:ALTER TABLE user2 DROP tmp –>컬럼 삭제

DROP

DROP DATABASE <데이터베이스 이름>

DROP TABLE <데이터베이스 이름>

DATA TYPE

정수타입: TINYINT, SMALLINT, MEDIUMINT, INT, BIGINT

실수 타입: FLOAT

날짜: DATE(날짜), DATETIME(년-월-일 시:분:초), TIMESTAME(DATETIME과 차이점은 날짜를 입력하지 않으면 현재 날짜와 시간을 자동으로 저장한다), TIME(시분초), YEAR

문자열: CHAR, VARCHAR, TEXT

CONSTRAINT

데이터 베이스의 테이블을 생성할때 각 컬럼은 각각의 제약조건을 갖는다

NOT NULL: 비어있는 값을 저장

UNIQUE: 같은 값을 저장할 수 없다

PRIMARY KEY: NOT NULL 과 UNIQUE 의 제약조건을 동시에 만족해야한다. 그러므로 컬럼에 비어 있는 값과 동일한 값을 저장할 수 없다. 하나의 테이블에 하나의 컬럼만 PRIMARY KEY 조건을 설정할 수 있다

FOREIGN KEY: 다른 테이블과 연결되는 값이 저장된다

DEFAULT: 데이터를 저장할때 해당 컬럼에 별도의 저장값이 없으면 DEFAULT로 설정된 값이 저장된다

AUTO_INCREMENT: 주로 테이블의 PRIMARY KEY 데이터를 저장할 때 자동으로 숫자를 1씩 증가시켜 주는 기능으로 사용한다

INSERT

테이블 이름 뒤에 오는 컬럼이름은 생략이 가능하며 대신에 VALUES 뒤에 value값이 순서대로 와야한다

INSERT INTO <테이블이름>(column_name_1, column_name_2 ... : optional) VALUES(value1, value2 ...)

UPDATE…SET

업데이트시에는 항상 select-where 로 변경할 데이터를 확인하고 update해줘야 실수를 줄일수 있다. 또한 limit도 함께 사용해주면 좋다.
