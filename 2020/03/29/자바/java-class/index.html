자바 "클래스" 란?

2020-03-29
Develop, Java

자바 클래스

생성자 오버로딩: 생성자는 클래스에 하나만 만들 수 있는게 아니라 여러개 만들수 있다. 단 모든 생성자들이 설도 다른 매개변수 개수 혹은 매개변수 데이터 타입을 가지고 있어야 한다

위와 같이 같은 생성자 이름을 갖지만 매개변수의 데이터 타입이 다른거나 매개변수의 개수가 다를경우 이를 생성자 오버로딩 이라고 부른다

생성자 오버로딩의 한가지 문제점은 생성자마다 필드를 초기화 하는 내용을 써주는것이 엄청 번거로운 것이다. 이를 해결하기 위해 생성자 하나만 필드를 초기화 하는 내용을 가지고있고 다른 생성자에서 필드 초기화 정보를 가지고있는 생성자를 호출 하면된다. 이때 this() 함수를 호출한다

this() 를 사용하면 필드초기화를 가지고 있는 생성자 함수를 호출할 수 있다.

클래스내에 같은 이름의 메소드를 여러개 선언하는 것을 메소드 오버로딩 이라고 한다. 생성자 오버로딩과 동일하게 매개변수의 개수와 매개변수의 타입이 다르면 오버라이딩이다.

static

자바스크립트의 prototype과 동일

모든 객체마다 자기자신의 값을 가지고 있어야 할 경우 인스턴스 필드와 인스턴스 메소드 를 갖으면 된다.

모든 객체가 하나의 공통된 값을 가져야 할 경우 정적 필드와 정적 메소드를 갖으면 를 갖으면 된다.

static은 객체마다 공통된 값을 가져야 할때 사용한다.

인스턴스 멤버와 인스턴스 메소드는 객체마다 값을 가져야 할 경우 사용한다

정적 변수는 선언과 초기화를 동시에 할 수 있다.

정적 블록은 클래스가 메모리로 로딩될 때 자동적으로 실행된다. 그렇기 때문에 정적 블록과 정적 메소드 내부에서 인스턴스 필드를 사용할 수 없다. static 블록이 클래스가 생성과 동시에 실행되기 때문에 인스턴스 멤버와 인스턴스 함수를 내부에서 사용할 수 없다.

싱글톤 (Singleton)

가끔 전체 프로그램에 단 하나의 객체만 만들도록 보장해야 하는 경우가 있다. 싱글톤을 만들려면 클래스 외부에서 new 연산자로 생성자를 호출할 수 없도록 막아야 한다. 생성자를 호출할 만큼 객체가 생성되기 때문이다.

생성자를 외부에서 호출하는것을 막기 위해서는 private 접근 제한자를 이용해야한다

Final 필드와 상수

final 필드는 쵝값이 저장되면 이것이 최종적인 값이 되어서 프로그램 실행 도중에 수정 할수 없다는 뜻이다. 자바스크립트의 const 와 마찬가지로 한번 지정되면 그이후로 변경할 수 없게된다(하지만 자바에서 final 을 상수라고 하지 않는다.)

상수

final 와 상수는 둘다 마찬가지로 한번 초기화 되면 수정할 수 없게 된다. 하지만 자바에서 final 을 상수라고 부르지 않는다. 그 이유는 불변의 값은 객체마다 저장할 필요가 없이 하나의 값이 공유되는 형식이기 때문이다. final 필드는 객체 마다 저장되고, 생성자의 매개값을 통해서 여러 가지 값을 가질 수 있기 때문에 상수가 될 수 없다. 자바에서 상수는 static 이면서 final 이어야 한다.

접근제한자 (Access Modifier)

접근 제한자의 종류는 다음과 같다: public, protected, default, private

접근제한자 적용대상 접근할수 없는 클래스
public 클래스, 필드, 생성자 메서드 없음(어디서든 사용 가능)
protected 필드, 생성자, 메서드
default 클래스, 필드, 생성자, 메서드 다른 패키지에 소속된 클래스(같은 패키지에서는 모두 접근 가능)
private 필드, 생성자, 메서드 모든 외부 클래스

class 의 접근 제한

default 접근 제한자: 클래스를 선언할 때 public을 생략하면 default 접근 제한을 가진다. 클래스가 default 접근 제한을 가지게 되면 같은 패키지에서는 아무런 제한 없이 사용할 수 있지만, 다른 패키지에서는 사용할 수 없도록 제한된다

public 접근 제한자: 클래스를 선언할 때 public 접근 제한자를 사용하면 클래스는 public 접근 제한을 갖는다. 클래스가 public 접근 제한을 가지게 되면 같은 패키지 뿐만 아니라 다른 패키지에서도 아무런 제한없이 해당 클래스를 사용할 수 있다.

생성자의 접근 제한

어떤 접근 제한자를 클래스에서 사용하냐에 따라 생성자 함수가 다른게 동작한다.

접근제한자 설명
public public 접근 제한은 모든 패키지에서 아무런 제한 없이 생성자를 호출할 수 있도록 한다. 생성자가 Public 접근 제한자를 가진다면 클래스도 public 접근 제한을 가지는게 정상적이지다. 클래스가 default 접근 제한ㅇ르 가진다면 클래스 사용이 같은 패키지로 한정되므로, 비록 생성자가 public 접근 제한을 가지더라도 같은 패키지에서만 생성자를 호출할 수 있다. 생성자의 접근 제한자가 public일 경우 클래스의 접근제한자에 맞춰서 생성자를 호출할 수 있다.
protected protected 접근 제한은 default 접근 제한과 마찬가지로 같은 패키지에 속하는 클래스에서 생성자를 호출 할 수 있다. 차이점은 다른 패키지에 속한 클래스가 해당 클래스의 자식 클래스라면 생성자를 호출할 수 있다.
default 생성자를 선언할 때 public 또는 Private를 생략하게 되면 자동을 default 접근 제한자를 갖게 된다. default접근 제한은 같은 패키지에서는 아무런 제한없이 생성자를 호출할 수 있지만, 다른 패키지에서는 생성자를 호출할 수 없다.
private private 접근 제한은 동일한 패키지이건 상관없이 생성자를 호출하지 못하도록 제한한다. 따라서 클래스 외부에서 new 연산자로 객체를 생성할 수 없다. 오로지 클래스 내부에서만 생성자를 호출할 수 있고, 객체를 만들 수 있다 Getter, Setter.

필드와 메소드의 접근 제한

필드와 메소드를 선언할 때 고려해야 할 사항은 클래스 내부에서만 사용할 것인지, 패키지 내에서만 사용할 것인지, 아니면 다른 패키지에서도 사용할 수 있도록 할 것인지를 결정해야한다.

Getter와 Setter

일반적으로 객체지향 프로그래밍 에서 객체의 데이터는 객체 외부에서 직접적으로 접근하는 것을 막는다. 그 이유는 객체의 데이터를 외부에서 마음대로 읽고 변경할 경우 객체의 무결성이 깨어질 수 있기 때문이다. 예를들어 자동차의 속도가 음수가 될수 없는데 외부에서 음수로 변경하면 객체의 무결성이 깨진다. 이러한 문제점을 해결하기 위해 객체 지향 프로그래밍에서는 메소드를 통해서 데이터를 변경하는 방법을 선호하는데, 이러한 메서드를 Getter 와 Setter라고 부른다. Getter 와 Setter 에서 조건을 주어서 객체의 무결성이 깨지는것을 막을 수 있다.

클래스를 선언할 때 가능하다면 필드를 private로 선언해서 외부로부터 보호하고, 필드에 대한 Setter와 Getter 메소드를 작성해서 필드값을 안전하게 변경/사용하는것이 좋다.

Annotation

어노테이션은 메타데이터 라고 볼 수 있다. 메타데이타란 애플리케이션이 처리해야 할 데이터가 아니라, 컴파일 과정과 실행 과정에서 코드를 어떻게 컴파일하고 처리할 것인지를 알려주는 정보이다.

@어노테이션이름

어노테이션의 3가지 용도:

  • 컴파일러에게 코드 문법 에러를 체크하도록 정보를 제공
  • 소프트웨어 개발 퉁이 빌드나 배치 시 코드를 자동으로 생성할 수 있도록 정보를 제공
  • 실행 시(런타임 시) 특정 기능을 실행하도록 정보를 제공

컴파일러에게 코드 문법 에러를 체크하도록 정보를 제공하는 대표적인 예는 @Override 어노테이션이다. @Override는 메소드 선언시 사용 하는데, 메소드가 오버라이드된 것임을 컴파일러에게 알려주어 컴파일러가 오버라이드 검사를 하도록 해준다. 정확히 오버라이드가 되지 않았다면 컴파일러는 에러를 발생시킨다. 어노테이션은 빌드시 자동으로 XML 설정 파일을 생성하거나, 배포를 위해 JAR압출 파일을 생성하는데에도 사용된다.

자바에서 기본적으로 제공하는 Annotation 은 다음과 같다: @Deprecated, @Override, @SuppressWarnings

@Deprecated: 해당 클래스/메소드 등은 더이상 지원하지 않으니 or 만들고 나니깐 별로 안좋은 부분이 많거나, 더 좋은 해결법이 생겼으니 사용하지 말라는 뜻이다. 그럼에도 불구하고 해당 메소드/클래스를 사용하면 자바 컴파일러는 경고 메세지를 날려준다.

@Override: 해당 Annotation 은 특정 클래스가 Superclass에 의해 오버라이딩 됬다는 것을 의미한다. 만약 메서드가 슈퍼클래스와 매칭되지 않으면 에러 메세지를 반환한다.

@SuppressWarnings: 해당 메소드에 대해서 compiler suppress warnings(컴파일러 경고 억제)를 생성한다. 쉽게말해 경고메세지를 생략하는 것이다.
