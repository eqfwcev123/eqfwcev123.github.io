상속 추가 설명

상속을 해도 부모 클래스의 모든 필드와 메소드를 물려받는 것은 아니다. 부모 클래스에서 private 접근 제한을 갖는 필드와 메소드는 상속 대상에서 제외된다 (private 을 선언한 클래스 필드/메소드 는 클래스 밖에서는 사용할 수 없다).

부모 클래스와 자식 클래스가 다른 패키지에 존재한다면 default 접근 제한을 갖는 필드와 메소드도 상속 대상에서 제외된다.

상속할때의 접근제한자:

서로 다른 패키지 내부에서 자식 클래스가 다른 패키지에 존재하는 부모클래스를 상속 받을 경우, 부모 클래스의 필드/메소드가 protected, public일 때만 상속 받을 수 있다. private, default이면 상속 받을 수 없다.

같은 패키지 내부에서 부모 클래스의 필드와 메소드가 default, protected, public일경우 자식 클래스는 모두 상속을 받을 수 있다. 단 접근제한자가 private일 경우 상속을 받을 수 없다.

super()

자식 클래스가 생성되면서 부모 클래스를 상속 받을때, 부모클래스는 존재 해야한다. 쉽게 생각하면, 부모가 있어야 자식이 존재한다.

부모클래스는 자식 클래스가 생성될때 생성된다. 자식 객체가 생성될때 생성자를 호출 하는데, 자시클래스의 생성자에서 부모클래스의 생성자를 호출한다.

super()를 명시해주지 않으면 자바 컴파일러는 자동으로 부모의 생성자를 호출한다.

만약에 부모 클래스에 생성자를 지정해주지 않았으면 super() 는 디폴트 생성자를 호출하지만, 만약에 생성자 함수가 존재할 경우 해당 생성자함수의 매개변수및 매개변수타입에 맞체 super() 를 호출해야한다

super는 매가값의 타입과 일치하는 부모 생성자를 호출한다. 만약 매개값의 타입과 일치하는 부모 생성자가 없을 경우 컴파일 오류가 발생한다.

메소드 재정의

상속된 부모 클래스의 메소드를 재정의해서 자식 클래스에서 사용. 메소드 오버라이딩 이라고 부른다

메소드 재정의(@Override): 메소드 오버라이딩은 상속된 메소드의 내용이 자식 클래스에 맞지 않을 경우. 자식 클래스에서 동일 한 메소드를 재정의하는 것을 의미한다. 메소드가 오버라이딩되었다면 부모 객체의 메소드는 숨겨지기 때문에, 자식 객체에서 메소드를 호출하면 오버라이딩된 자식 메소드가 호출된다.

메소드 오버라이딩을 할때는 몇가지 규칙이 있다.

  • 부모의 메소드와 동일한 리턴타입, 메소드 이름, 매개변수 리스트 를 가져야 한다
  • 접근 제한을 더 강하게 오버라이딩 할 수 없다(public에서 private혹은 default 로 변경 불가)
  • 새로운 예외를 throws할수 없다

접근 제한을 더 강하게 할 수 없다는 것은 부모 메소드가 public 접근 제한을 가졌을때 오버라이딩 하는 자식 메소드는 default 나 private 접근 제한으로 수정할 수 없다는 뜻이다. 하지만 그 반대는 가능하다

부모 메소드 호출(super)

자식 클래스에서 부모 클래스의 메소드를 오버라이딩 하게되면, 부모 클래스의 메소드는 숨겨지고 오버라이딩 된 자식 메소드만 사용하게 되는것이다. 그러나 자식 클래스 내부에서 오버라이딩된 부모 클래스의 메소드를 호출해야 하는 상황이 발생하면 super.부모메소드를 호출하면 된다.

super() 는 부모의 생성자 호출

super.부모메소드() 는 부모의 메소드를 호출

상속할 수 없는 final 클래스

final 키워드를 사용하게 될경우 더이상 값을 새로 할당할 수 없다고 했는데 메소드와 클래스에 final 을 지정할 경우 상속과 관련이 있다.

클래스를 서언할 때 final 키워드를 class앞에 붙이게 되면 이 클래스는 최종적인 클래스이므로 상속할 수 없는 클래스가 된다. 즉 final클래스는 부모 클래스가 될 수 없고 자식 클래스를 갖을 수 없다

public final class 클래스명{...}

자바의 String 클래스도 final 클래스 이기 때문에 자식 클래스를 갖을 수 없다.

오버라이딩 할수 없는 final 메소드

메소드를 선언할 때 final 키워드를 붙이게 되면 이 메소드는 최종적인 메소드이므로 오버라이딩 할 수 없는 메소드가 된다.

메소드에 final이 붙을 경우 오버라이딩 할 수 없다.

즉 부모 클래스를 상속해서 자식 클래스를 선언할 때 부모클래스에 선언된 final 메소드는 오버라이딩 할 수 없다.

public final 리턴타입 메소드명(){}

상속에서의 protected 접근 제한

protected 는 public 와 default 접근 제한자의 중간이라고 생각하면 된다. 같은 패키지에서는 default 와 같이 접근 제한이 없지만, 다른 패키지에서는 자식 클래스만 접근을 허용한다.

protected는 필드와 생성자, 메소드 선언에 사용될 수 있다.

타입 변환과 다형성

다형성은 같은 타입이지만 실행 결과가 다양한 객체를 이용할 수 있는 성질을 말한다. 코드의 측면에서 보면 다형성은 하나의 타입에 여러 객체를 대입합으로써 다양한 기능을 이용하 수 있도록 해준다. 다형성을 위해 자바는 부모 클래스로 타입 변환을 허용한다. 즉 븀ㅎ 타입에 모든 자식 객체가 대입될 수 잇다. 이것을 이용하면 객체는 부품화가 가능해진다.

예를들어 자동차 타이어에 금호 타이어와 현대 타이어가 있고, 둘다 타이어라는 클래스로부터 상속을 받을경우 다음같이 다형성을 구현할 수 있다.

타입 변환이란 데이터 타입을 다른 데이터 타입으로 변환하는 행위를 말한다. 클래스 타입의 변환은 상속관계에 있는 클래스에서 발생한다. 자식 타입은 부모 타입으로 자동 타입 변환이 가능하다.

자동 타입 변환(Promotion): 자동 타입 변환의 개념은 자식은 부모의 특징과 기능을 상속받기 때문에 부모와 동일하게 취급될 수 있다.

부모클래스 변수 = 자식클래스타입; (자동 타입 변환)

예를들어 고양이는 동물의 특징과 기능을 상속받는다. 그렇기 때문에 고양이는 동물이다 라는 말이 설립된다. 이것을 코드로 해석하면 다음과 같다.

자동 타입 변환은 부모가 아니더라도 상속 계층에서 상위 타입이면 자동 타입 변환이 일어날 수 있다.

이런식으로 계층 구조가 있을때 자동 타입 변환은 다음같이 동작한다

B 타입의 참조변수에 e 인스턴스 객체를 못넣는 이유는 상속 관계가 아니기 때문이다.

부모 타입으로 자동 타입 변환된 이후에는 부모 클래스에 선언된 필드와 메소드만 접근 할 수 있다. 비록 변수는 자식 객체를 참조하지만 변로 접근 가능한 멤버는 부모 클래스 멤버로만 한정된다. 그러나 예외가 있는데, 메소드가 자식 클래스에서 오버라이딩 되었다면 자식 클래스의 메소드가 대신 호출된다.

다형성 예제:

필드의 다형성

다향성이란 동일한 타입을 사용하지만 다양한 결과가 나오는 성질을 말한다. 주로 필드의 값을 다양화함으로서 실행 결과가 다른게 나오도록 구현하는데, 필드의 타입은 변함이 없지만, 실행 도중에 어떤 객체를 필드로 저장하느냐에 따라 실행 결과가 달라질 수 있다.

타입 캐스팅

다형성에서 자식 타입이 부모타입으로 자동 변환 한다고 했다. 반대로 부모타입이 자식 타입으로 다운 캐스팅 될 수 있다. 다운 캐스팅은 항상 되는것이 아니라, 자식타입이 부모타입으로 형변환 후, 다시 자식 타입으로 변환을 할려고 할때 사용할 수 있다.

자식클래스 변수 = (자식 클래스) 부모클래스타입;

객체 타입 확인하기 (instnace of)

한 객체가 다른 객체의 자식인지 알기위해 사용

좌항(객체) instanceof 우항(타입)

추상 클래스

추상의 사전적 의미는 실체 간에 공통되는 특성을 추출하는 것이다. 예를들어 새, 곤충, 물고기등의 실체에서 공통되는 특성을 추출해보면 동물이라는 공통점이 나온다. 또다른 예시로 삼성, 현대, LG 라는 실체에서 공통되는 특성을 추출하면 회사라는 공통점이 나온다.

클래스에서도 추상 클래스가 존재한다. 객체를 직접 생성할 수 있는 클래스를 실체 클래스라고 하면(우리가 평소에 사용하는 클래스) 이 클래스들의 공통적인 특성을 추출해서 선언한 클래스를 추상 클래스라고 부른다.

추상 클래스와 실체 클래스는 서로 상속을 관계를 갖는다. 추상 클래스가 부모이고, 실체 클래스가 자식으로 구현되어 실체 클래스는 추상 클래스의 모든 특성을(메소드, 필드) 물려받고, 추가적인 특성을 가질 수 있다.

예를들어 Bird.class, Insect.class, Fish.class 등의 실체 클래스에서 공통되는 필드와 메소드를 따로 선언한 Animal.class 클래스를 만들 수 있다.

추상 클래스는 실체 클래스의 공통되는 필드와 메소드를 추출해서 만들었기 때문에 객체를 직접 생성해서 사용할 수 없다. 다시 말해서 추상 클래스는 new를 이용해서 객체를 생성할 수 없다(장고모델에서 사용한것 처럼 필드만 공유하는 형식)

Animal animal = new Animal(); // X

class Ant extends Animal{ ... } // O

추상클래스를 사용하는 이유

실체 클래스의 공통적인 특성(필드와 메소드)를 뽑아내어 추상 클래스로 만드는 두가지 이유:

  1. 실체 클래스들의 공통된 필드와 메소드의 이름을 통일할 목적

    실체 클래스를 작성하는 사람이 여러사람일 경우, 실체클래스 마다 필드와 메소드 이름을 제각각 작성할 수 있다. 이를 방지하기 위해서 추상 클래스에 모든 실체 클래스들이 사용할 공통된 필드와 메소드의 이름을 작성 해놓고, 실체 클래스에서 추상 클래스를 상속해서 네이밍을 그대로 사용하면 된다.

  2. 실체 클래스를 작성할 때 시간 절약

추상 클래스 사용 방법

추상 클래스를 선언할 때에는 선언에 abstract 키워드를 붙이면 된다. abstract 키워드를 붙이게 되면 new 연산자를 이요해서 객체를 생성하지 못하게 되고, 오로지 상속에 의해서 필드와 메소드를 상속받아서 사용할 수 있게 된다.

public abstract class 클래스이름{ 필드, 생성자, 메소드 }

자식 클래스에서 super()를 이용해서 생성자 호출. new를 사용하지 않는다.

추상 클래스도 일반 클래스와 마찬가지로 필드, 생성자, 메소드 선언을 할 수 있다. new 연산자로 생성자를 집접 호출할 수 없지만 자식 객체가 super() 를 호출해서 추상 클래스의 생성자 함수를 호출할 수 있다.

추상 메서드와 오버라이딩

추상 클래스는 실체 클래스가 공통적으로 가져야 할 필드와 메소드들을 정의해 놓은 추상적인 클래스이므로 실제 클래스의 멤버(필드, 메소드)를 통일화 하는데 목적이 있다. 모든 실체들이 가지고 있는 메소드의 실행 내용이 동일하다면 추상 클래스에 메소드를 작성하는것이 좋다. 하지만 메소드의 선언만 통일화하고, 실행 내용은 실체 클래스마다 달다랴 하는 경우가 있다. 예를들어 모든 동물들의 울음소리를 sound()라는 함수로 지정을 한다고 한다고 할때, 이런 함수는 개발자 직접 일일히 입력 해줘야 하는 메소드 이기 때문에 선언부만 추상 클래스에서 하고 실체 클래스에서 오버라이딩을 해줘야 한다.

이런 경우를 위해서 추상 클래스 내부에서 추상 메서드 를 선언할 수 있다. 추상 메서드는 추상 클래스에서 만 선언할 수 있는데, 메소드는 선언부만 있고 실행 내용인 중괄호{} 는 존재하지 않는다.
