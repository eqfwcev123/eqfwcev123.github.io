Boto3 Credentials

Boto 는 다양한 방법으로 환경설정을 할 수있다. 그중 AWS Credentials 와 region은 꼭 가지고 있어야 한다.

Boto3설정을 할때 두가지 데이터가 존재한다: credential data 와 non-credential data 두가지이다.

Credential data는 aws_access_key_id, aws_secret_access_key, aws_session_token 와 같은 데이터가 있고, non-credential data 에는 region 정보가 있다.

Credential data 와 non Credential데이터에 의해 검색 과정이 달라질 수 있기 때문에 두가지 데이터를 잘 구분할 줄 알아야 한다. Boto3는 non-credenital data를 찾을때와는 다르게 credential data를 찾을때는 다양한 위치에서 credential data를 찾는다.

boto3가 credential 데이터를 찾을때 credential data가 있을만한곳을 모두 검색을 하는데, 검색을 하는 순서는 다음과 같다:

  1. boto.client() 메서드로 전달된 매개변수에 있는 credential 데이터
  2. Session 객체를 생성할 때 credential을 매개변수로 전달
  3. 환경 변수
  4. 공유되는 credential 파일 (~/.aws/credentials)
  5. AWS config파일(~/.aws/config)
  6. Assume Role Provider
  7. Boto2 config file(/etc/boto.cfg and ~/.boto)
  8. Amazon EC2(IAM role 보유)에 있는 인스턴스 Metadata 서비스

1. Client(), Session()메소드의 파라미터로 credential data 전달

client 혹은 session을 생성할 때 매개변수로 credential data를 전달하는 방법이다.

ACCESS_KEY, SECRET_KEY_SESSION_TOKEN 에 사용자의 접근키를 그대로 입력하는 방법을 추천하지 않는다.

2. 환경 변수

boto3는 환경 변수에서 AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_TOKEN, AWS_SESSION_TOKEN 을 찾는다.

3. shared Credential 파일

credentials 파일은 기본적으로 ~/.aws/credentials 라는 경로에 지정되어있다. 만약에 credentials 파일의 default위치를 변경하고 싶을 경우 AWS_SHARED_CREDENTIALS_FILES 환경 변수를 변경하면 된다.

shared credential 파일은 다음과 같다 :

[default]를 profile name 이라고 부르는데, profile name을 다른 이름으로 변경해도 된다. profile name을 변경한 후 다음과 같이 사용하면 된다.

session에 profile_name을 불러와서 해당 세션으로 client 를 생성할 수 있다.

4. Assume Role Provider

AWS_CONFIG_FILE 환경 변수에 명시된 파일이나, 기본 경로인 ~/.aws/config 파일에서 role에 대한 정보를 명시하여, boto3가 자동으로 AWS STS에 AssumeRole을 호출하도록 할수도 있다. role_arn, source_profile, role_session_name 등의 파라미터가 필요하다.

5.boto2 Config

Boto3 는 boto2 config파일에서 credentials 를 로드 할려고한다. boto3는 먼저 BOTO_CONFIG환경 변수에 지정되어 있는 파일을 체크하고 없을 경우 /etc/boto.cfg파일과 ~/.boto를 체크한다. ~/.boto파일에서 [Credentials]영역만 사용이 된다.

6. IAM Role

Amazon EC2에서 실행 중이며 위의 방법들로 자격 증명을 찾지 못한 경우, boto3는 EC2 인스턴스의 메타 데이터 서비스에서 자격 증명을 로드한다. EC2 인스턴스를 시작할 때 IAM role을 지정해 두었다면, 별도의 명시적인 구성을 하지 않아도 boto3가 알아서 인스턴스의 자격 증명을 사용한다.

Keywords

  1. Session: 세션은 상태를 관리한다. boto3 의 세션을 사용하면 client 와 resource를 생성할 수 있다. 기본적으로 세션은 사용자가 필요할때 생성되지만, 사용자가 직접 session을 관리하는 것을 추천한다.

    • 기본적으로 boto3는 default session의 역할을 해준다(자동으로 세션을 생성해준다.)
    • s3 = boto3.resource('s3'). (boto3 가 default 로 생성해준 session)
    • s3 = boto3.client('s3')
    • 사용자가 직접 세션을 관리하기 위해서는 session = boto3.session.Session()을 이용할 수 있다.

  2. client

    • low-level 인터페이스
    • AWS Service description에 의해 생성
    • botocore 수준의 client를 공개(botocore 는 AWS cli 와 boto3의 기초가 되는 라이브러리이다)
    • AWS 서비스 API와 1:1로 매핑한다

  3. Resource

    • high-level, 객체지향적 인터페이스
    • resource description에 의해 생성
    • 식별자(identifier)와 속성(attribute)를 사용

  4. Credentials
