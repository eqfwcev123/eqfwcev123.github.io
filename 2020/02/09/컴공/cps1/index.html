운영체제의 역할: 시스템 자원(System Resource) 관리자

운영체제의 역할: 시스템 자원(System Resource) 관리자

Operating System 또는 OS 라고 부른다

여기서 시스템 자원이란? 컴퓨터 하드웨어

–> CPU(중앙처리장지), Memory(DRAM, RAM)

–> I/O Devices(입출력 장치) : Monitor, Mouse, Key Board etc

–> 저장매체: SSD, HDD(하드디스크)

​ 컴퓨터 하드웨어는 스스로 할수 있는 것이 하나도 없다

운영체제가 해주는 역할들:

  1. CPU: 각 프로그램이 얼마나 CPU를 사용할 지를 결정할 수 없다.
  2. Memory: 각 프로그램이 어느 주소에 저장되어야 하는지, 어느정도의 메모리 공간을 확보 해줘야 하는지를 결정할 수 없다.
  3. 저장매체(HDD, SSD): 어떻게, 어디에 저장할지는 결정할 수 없다
  4. I/O (키보드/마우스): 키보드를 입력하면 어떤 단어가 나와야 하고, 마우스를 움직이면 마우스 포인터가 어느정도 움직이는지를 운영체제가 결정 해준다

대표적은 운영 체제들

  • windows OS, Mac OS, Unix 계열 OS(Unix 와 사용법이나 OS 구조가 유사한 OS): 리눅스

운영체제의 역할2: 사용자와 컴퓨터 간의 소통

  • shell

운영체제의 역할3: 컴퓨터 하드웨어와 응용 프로그램을 제어

  • Process management
  • Main Memory Management
  • File Management
  • Disk Management
  • I/O Management
  • Networking, Protection

응용 프로그램(Application)

  • 프로그램 = 소프트웨어

  • 소프트웨어 = 운영체제, 응용프로그램(엑셀, 파워포인트, 우리가 만드는 프로그램)

  • 응용프로그램은 운영체제 위에서 동작을 한다

운영체제와 응용 프로그램 간의 관계

  • 운영 체제는 응용 프로그램 을 관리한다. 여기서 관리의 뜻은:
    • 응용 프로그램을 실행시킨다
    • 응용 프로그램 간의 권한을 관리한다(관리자 권한 실행)
    • 응용 프로그램ㅁ을 사용하는 사용자도 관리 (ex: 컴퓨터 로그인)
  • 시스템 자원을 관리한다
  • 사용자와 컴퓨터간의 커뮤니케이션을 지원

1960년대 초반

Batch processing system: 여러 응용 프로그램을 등록 시켜 놓으면, 순차적으로 실행하는 시스템(애플리케이션을 등록 해두면 순차적으로 하나씩 실행). 첫번재 application 이 끝나야 두번째 application을 실행 시킬 수 있다.

배치 처리 시스템을 기반으로 운영체제가 처음 출현을 하게 된다.

1960년대 후반

  • 새로운 개념이 제안됨

    • 시분할 시스템(Time Sharing System)

    • 멀티 태스킹

      ​ 운영 체제로 구현되지 는 않았다.

시분할 시트템과 멀티 태스킹: 응용 프로그램이 CPU를 사용하는 시간을 잘개 쪼개서, 여러 개의 응용 프로그램을 동시에 실행하는 기법

  • 시분할 시스템과 멀티 태스킹의 공통점과 차이점:
    • 공통점: 둘다 시간을 쪼개서 사용
    • 차이점:
      • 시분할 시스템: 다중 사용자를 지원하고, 컴퓨터 응답 시간을 최소화 하는 시스템
      • 멀티테스킹: 단일 CPU,에서 여러 응용프로그램의 병렬 실행을 가능케 하는 시스템

응용 프로그램, 운영 체제, 컴퓨터 하드웨어 관계

  • 운영 체제는 응용 프로그램이 요청하는 메모리를 허가하고 분배한다
  • 운영 체제는 응용 프로그램이 요청하는 CPU시간을 제공한다
  • 운영 체제는 응용 프로그램이 요청하는 IO Devices사용을 허가/제어한다

운영 체제는 사용자 인터페이스를 제공 한다

  • Shell, Zsh 등등 프로그램이 인터페이스를 제공한다
    • 사용자가 운영체제 기능과 서비스를 조작할 수 있도록 인터페이스를 제공하는 프로그램

운영 체제는 응용 프로그램을 위해서도 인터페이스를 제공한다

  • Application Programming Interface(API): 함수
  • 함수들의 묶음을 라이브러리라고 부른다

System call

system call is the programmatic way in which a computer program requests a service from a kernel of the operating system it is executed on. A System call is a way for programs to interact with the *operating system *.

시스템콜:

  • 시스템 콜 또는 시스템 호출 인터페이스 라고 부른다
  • 운영 체제가 운영 체제 각 기능을 사용할 수 있도록 시스템 콜이라는 명령 또는 함수를 제공
  • API 내부에는 시스템콜을 호출하는 형태로 만들어 지는 경우가 대부분이다.

운영 체제 구조

시스템콜은 OS를 조작할 수 있는 기능들. API는 시스템 콜을 사용하는 기능들.

시스템 콜==> 별도 함수

라이브러리/API ==> 시스템콜을 사용하는 별도의 함수

CPU protection Rings(권한의 차이)

각 CPU마다 Protection Ring 의 개수가 다를 수 있다.

CPU 도 권한 모드라는 것을 가지고있다. 주로 사용되는 두가지 모드: 사용자 모드 와 커널 모드

kernel 이란? OS 커널(OS 핵심기능)

  • 사용자 모드(user mode)/ 애플리케이션이 CPU를 사용하는 모드
  • 커널 모드(Kernel mode)/OS 가 CPU를 사용하는 모드: 특권 명령어 실행과 원하는 작업 실행을 위한 자원 접근을 가능케 하는 모드.

시스템 콜은 커널 모드로 실행

  • 커널 모드에서만 실행 가능한 기능들이 있다
  • 커널 모드로 실행 하려면, 반드시 시스템 콜을 거쳐야 한다(사용해야 한다)
  • 시스템콜은 운영체제가 제공한다

사용자 모드와 커널모드

  • 함부로 응용 프로그램이 전체 컴퓨터 시스템을 헤지지 못하게 한다
  • 이것은 마치 주민등록본은 꼭 동사무소 또는 민원24시(정부 사이트)에서 특별한 신청서(시스템콜)를 써야만 발급 받을 수 있는것과 같다
    • 동사무소 분들은 특별 권한을 가지고, 주민등록본 출력 명령을 실행할 수 있다.

프로세스 스케쥴링

  • 프로세스를 운영 체제라고 생각하면 된다.

  • 스케쥴링 방법: 배치하는 방법(배치 처리 시스템, 시분할 시스템, 멀티프러그래밍 시스템 등등)

  1. 배치 처리 시스템: 자료구조의 Queue 와 비슷하다고 생각하면 된다 (First In First Out), 처음 등록되는 Application 이 실행되고, 다 실행되면 다음 Application 이 실행 된다.

배치 처리 시스템이 사용되지 않는 이유: 여러프로그램을 순차적으로 실행시킬 수 있도록 해주세요! 때문에 나온 시스템이다.

  • 단점: 어떤 프로그램은 실행 시키는 시간이 너무 오래 걸려서 다음 시스템이 동작하는데 소요되는 시간이 너무 크다!
  • 배치 처리 시스템에서 두개의 프로그램을 동시에 실행시킬 수 없다( 음악(멜로)을 들으면서 문서(워드)를 작성할 수 없다.)
  • 여러 사용자가 동시에 하나의 컴퓨터를 사요할 수 없다.

시분할 시스템

시분할 시스템:다중 사용자 지원을 위해 컴퓨터 응답 시간을 최소화 하는 시스템

멀티 태스킹

멀티 태스킹: 단일 CPU(한개의 코어)에서, 여러 응용 프로그램이 동시에 실행되는 것처럼 보이도록 하는 시스템

멀티 프로세싱: 여러개의 CPU. 여러 CPU에 하나의 프로그램을 병렬로 실행해서 실행속도를 극대화시키는 시스템

멀티 프로그래밍

목적: 최대한 CPU를 많이 활용 하도록 하는 시스템

  • 시간 대비 CPU활용도를 높이자
  • 응용 프로그램을 짧은 시간 안에 실행 완료를 시킬 수 있다.

응용 프로그램은 온전히 CPU를 쓰기보다, 다른 작업을 중간에 필요로 하는 경우가 많다

  • 예를들어 응용프로그램이 실행되는 도중에 파일을 읽는다(저장 매체). 저장 매체(SSD 등)는 cpu의 실행 시간으로 봐서는 엄청 느린 장치이다. 저장 매체에서 파일을 읽는 순간, 응용프로그램은 아무 작업도 못하게 되고, 그렇게 되면 cpu에 응용프로그램이 돌아가고 있다고 치더다로 더이상 실행을 할 수 없다(저장매체에서 파일을 읽기 전까지).

멀티 프로그래밍 사용전

CPU의 활용도는 3/8, 다시 말해 50% 이하이다

멀티 프로그래밍일 경우

CPU의 활용도는 8/8, 즉 100%이다.

메모리 계층

CPU에 하나의 명령어를 실행하는 시간 단위를 사이클 이라고 한다. 위 사진은 특정 지역에 있는 메모리에 접근하는 시간을 표시한다. 마지막에 Flash Drive(SSD) 와 Hard Disk(HDD) 같은 저장 매체에 넘어갈 경우 cycle수가 Cpu 보다 훨씬 높아진다(느려진다)

Register와 Level1, Level2, Level3 Cache는 모두 메모리 안에 존재한다

정리

실제로는 시분할 시스템, 멀티 프로그래밍, 멀티 태스킹이 유사한 의미로 통용된다.

  • 시분할 시스템: 다중 사용자 지원, 컴퓨터 응답 시간을 최소화 하는 시스템
  • 멀티 프로세싱: 여러 CPU에서 하나의 응용 프로그램을 병렬로 실행하게 해서, 실행 속도를 높이는 기법
