Posted on 2020-01-12 Edited on 2020-01-13 In Java

Package

패키지는 클래스의 묶음이다. 패키지에는 클래스 또는 인터페이스를 포함시킬 수 있으며, 서로 관련된 클래스들 끼리 그룹 단위로 묶어 놓음으로써 클래스를 효율적으로 관리할 수 있다. 같은 이름의 클래스 일지라도 서로 다른 패키지에 존재하는 것이 간으하므로, 자신만의 패키지 체계를 유지함으로써 다른 개발자가 개발한 클래스 라이브러리의 클래스와 이름이 충돌하는것을 방지할 수 있다.

클래스의 실제 이름은 클래스의 패키지 이름을 포함한 이름이다.

클래스가 물리적으로 하나의 .class 파일 인것과 마찬가지로, 패키지는 물리적으로 하나의 디렉토리이다. 그렇기 때문에 특정 패키지에 존재하는 하나의 클래스는 패키지 라는 디렉토리에 존재하는 클래스라고 생각하면 된다.

java.lang.String이라는 클래스는 java의 sub 디렉토리인 lang에 속하는 String.class 인 것이다. 다시말해 java 라는 패키지 내부에 lang 이라는 패키지가 있고, 그안에 String.class 가 있는 것이다.

package 패키지 이름 을 이용해서 패지를 선언할 수 있다.

import 문

모든 소스 파일에서 import 문은 package 다음 문에, 그리고 클래스 선언 문 이전에 위치해야 한다. 그리고 import문은 package 문과 달리 한 소스파일에 여러번 선언할 수 있다.

import 패키지명.클래스명 를 이용해서 import를 할 수 있다.

각 클래스 마다 static 한 멤버들이 있는데, 이런 정적 멤버들을 import하기 위해선느 import static 패키지.클래스명을 해줘야 한다.

오버라이딩

조상 클래스로부터 상속받은 메서드의 내용을 변경하는 것을 오버라이딩 이라고 한다.

부모 클래스한테서 상속 받은 메소드를 그대로 사용해도 되지만 자식 클래스 자신한테 맞춰서 메소드를 변경할 수 있다. 메소드를 오버라이딩 한다고 해서 다른 클래스들이 부모 클래스한테서 상속 받는 메소드도 변경 되는 것은 아니다.

Point3D 클래스는 Point 클래스를 상속 받는다. 하지만 Point3D 클래스 3Dimension 좌표를 반환 해야 하기 때문에 부모한테서 상속받은 메소드를 사용하면 안된다. 이때 부모한테서 상속받은 메소드를 오버라이딩 하면 된다. 상속받은 getLocation()메소드를 오버라이딩 해서 자기 자신의 새로운 메소드로 사용을 하면 x,y,z 세개의 좌표를 반환받을 수 있다.

오버로딩(Overloading)

메서드도 변수와 마찬가지로 같은 클래스 내부에서 구별될 수 있어야 하기 때문에 각기 다른이름을 가져야한다. 그러나 자바에서는 한 클래스 내에 이미 사용하려는 이름과 동일한 이름을 가진 메서드가 있더라도 매개변수의 개수, 매개변수 타입이 다르면 다른 메서드로 간주한다.

이처럼, 한 클래스 내부에 같은 메서드 이름을 갖지만 다른 동작을 하는 메서드를 메서드 오버로딩, 혹은 오버로딩 이라고 한다.

오버로딩을 사용하기 위한 조건

오버로딩을 하기 위해서는 다음과 같은 조건을 만족 시켜야한다.

  1. 같은 클래스 내부에 동일 한 이름을 갖은 메서드가 존재 해야한다
  2. 매개변수의 개수 또는 타입이 달라야한다

위 예제를 보면 People 이라는 클래스 내부에 두개의 People 디폴트 생성자 함수가 존재한다. 동일한 이름을 갖기 때문에 에러를 발생 시킬것 같지만, 둘은 다른 메서드 이다. 첫번째 생성자 함수는 두개의 매개변수(이름, 나이)를 입력해야 하지만 두번째 생성자 함수는 이름만입력을 하면 된다.

메소드 오버로딩을 사용하게 되면:

  1. 동일한 기능을 하는 메소드를 다른 이름으로 여러개를 만들 필요가 없다
  2. 메소드의 이름을 절약할 수 있다.

가변인자는 타입...변수명 과 같은 형식으로 선언한다.

클래스 메서드(static 메서드) 와 인스턴스 메서드

메서드 앞에 static 일 들어가면 해당 메서드는 static 메서드 이고 붙어 있지 않으면 인스턴스 메서드 이다. 클래스 메서드도 클래스변수처럼 객체를 생성하지 않고 ‘클래스이름.클래스메서드( )’ 형식으로 사용한다. 인스턴스 메서드는 인스턴스 변수와 마찬가지로 인스턴스를 생성해야 인스턴스 메서드를 ‘인스턴스.인스턴스메서드( )’ 형식으로 사용할 수 있다.

인스턴스 메서드는 인스턴스 변수와 관련된 작업을 하는, 즉 메서드의 작업을 수행하는 데 인스턴스를 필요로 하는 메서드 이다. 그런데 인스턴스 변수는 인스턴스를 생성해야하만 만들어 지므로, 인스턴스 메서드 역시 인스턴스를 생성해야만 호출 해야한다.

반면에 메서드 중에서 인스턴스와 관계없는 메서드를 클래스 메서드(정적 메서드) 로 정의한다.

  • 클래스를 설계할 때, 멤버변수 중 모든 인스턴스에 공통적으로 사용해야하는 것에 static을 붙인다.

    • 생성된 각 인스턴스는 서로 독립적이기 때문에 각 인스턴스의 변수는 서로 다른 값을 유지한다. 그러나 모든 인스턴스에서 같은 값이 유지되어야 하는 변수는 static 으로 선언을 하면 된다.

    • 클래스 변수(static 변수) 는 인스턴스를 생성하지 않아도 사용할 수 있다.

      • static이 붙은 변수는 클래스가 메모리에 올라갈 때 이미 자동적으로 생성되기 때문이다.

    • 클래스 메서드는 인스턴스 변수를 사용할 수 없다.

      • 인스턴스 변수는 인스턴스가 반드시 존재햐아만 사용할 수 있다. 클래스 메서드는 인스턴스 생성 없이 호출 가능하므로 클래스 메서드가 호출되었을 때 인스턴스가 존재하지 않을 수 있다. 그래서 클래스 메서드에서 인스턴스 변수의 사용을 금지한다. 반면에 인스턴스 변수나 인스턴스 메서드에서는 static이 붙은 멤버들을 사용하는 것이 언제나 가능하다. 인스턴스 변수가 존재한다는 것은 static 변수가 이미 메모리에 존재한다는 것을 의미한다.

        ​ 앞에서 말한것 처럼 클래스 파일이 사용이 되면 JVM 은 해당 .class 파일을 읽어서 분석하여 클래스에 대한 정보를 메소드 영역에 저장을 한다. 이때 클래스의 정적 메소드와 정적 멤버 변수가 최 우선적으로 생성이 된다.

    • 메서드 내에서 인스턴스 변수를 사용하지 않는다면 static을 붙이는 것을 고려한다.

      • 메서드의 작업내용 중에서 인스턴스 변수를 필요로 한다면, static을 붙일 수 없다. 반대로 인스턴스 변수를 필요로 하지 않는다면 static을 붙이는 것이 좋다. 메서드 호출 시간이 짧아지므로 성능이 향상된다. static을 안붙인 메서드(인스턴스 메서드)는 실행시 호출되어야할 메서드를 찾는 과정이 추가적으로 필요하기 때뭉네 시간이 더 걸린다.

자바의 변수와 메서드

변수의 종류 선언위치 생성 시기
클래스 변수 클래스 영역 클래스가 메모리에 올라갈 때
인스턴스 변수 클래스 영역 인스턴스가 생성될 때
지역 변수 클래스 영역 이외의 영역(메서드, 생성자, 초기화 블럭 내부) 변수 선언문이 수행될 때

인스턴스 변수

클래스의 영역에 선언되며, 클래스의 인스턴스를 생성할 때 만들어진다. 그렇기 때문에 인스턴스 변수의 값을 읽어 오거나 저장하기 위해서는 먼저 인스턴스를 생성해야한다. 인스턴스는 독립적인 저장 공간 을 가지므로 서로 다른 값을 가질 수 있다. 인스턴스마다 고유한 상태를 유지해야하는 속성의 경우 인스턴스 변수를 선언 한다

클래스 변수(static)

클래스 변수를 선언하는 방법은 인스턴스 변수 앞에 static을 붙이기만 하면 된다. 인스턴스마다 독립적인 저장공간을 갖는 인스턴스 변수와는 달리, 클래스 변수는 모든 인스턴스가 공통된 저장공간을 공유하게 된다. 한 클래스의 모든 인스턴스들이 공통적인 값을 유지해야 하는 속성의 경우, 클래스 변수를 선언해야한다.

클래스 변수는 인스턴스변수와 달리 인스턴스를 생성하지 않고도 사용할 수 있다. 하지만 인스턴스를 생성하지 않고 사용을 하기 때문에 클래스명.클래스변수 형식으로 사용한다. People 이라는 클래스가 있고, isHuman 이라는 클래스 변수가 있을 경우 People.isHuman 을 해서 클래스 변수를 호출 한다.

클래스가 메모리에 로딩될 때 생성되어 프로그램이 종료될 때 가지 유지되며, public 을 앞에 붙이면 같은 프로그램 내에서 어디서나 접근할 수 있는 전역 변수의 성격을 갖게된다.

지역 변수

메서드 내에 선언되어 메서드 내에서만 사용할 수 있는 변수이다. 메서드가 종료되면 소멸되어 사용할 수 없게 된다. for문 이나 if 문, 또는 while 문의 블록 내에 선언된 ({}로 감싸져 있는 함수혹은 문)변수들은 지역변수이며 해당 문이 종료될 때 내부에서 선언된 인스턴스 변수도 소멸한다.

인스턴스 변수는 인스턴스가 생성될 때 마다 생성되므로 인스턴스마다 각기 다른 값을 유지할 수 있지만, 클래스 변수는 모든 인스턴스가 하나의 저장 공간을 공유하므로 항상 공통된 값을 갖는다.

자바 상속(extends)

객체지향은 특징으로서 새로운 클래스를 정의 할 때 이미 구현된 클래스를 상속(inheritance) 받아서 속성이나 기능이 확장되는 클래스를 구현할 수 있다.

상속은 기존에 존재하는 클래스의 기능을 좀더 확장해서 사용하고 싶을때 사용한다

사용방법:

자식 클래스 extends 부모클래스`: 자식 클래스가 부모클래스한테서 상속을 받는다는 뜻이다.

  • 다중 상속을 할 수 없다: 클래스C 가 클래스A, 클래스B한테서 상속을 받을 수 없다.(부모 클래스는 무조건 한명). 자바는 single inheritance 만 허용한다

상위 클래스는 하위 클래스보다 일반적인 개념과 기능을 갖는다

하위 클래스는 상위 클래스보다 구체적인 개념과 기능을 갖는다

생성자함수와 초기화 블럭(클래스 초기화 블럭/인스턴스 초기화 블럭)은 상속되지 않는다

자식 클래스의 멤버 개수는 조상 클래스보다 항상 같거나 많다

접근제어자가 Private 이거나 default일 멤버들은 상속되지 않는다기 보다는 상속은 받지만 자식 클래스가 접근을 할 수없는것이다.

두개의 클래스(자식 클래스 A 와 자식 클래스 B)가 한개의 부모 클래스 한테서 상속을 받더라도 자식 클래스 A 와 B는 아무런 관계가 없다. 상속 관계에서 형제 관계는 없다.

하위 클래스는 상위 클래스의 기능을 마음대로 사용할 수 있지만, private 한 변수와 메소드는 사용할 수 없다. 만약에 외부 클래스는 내 private 변수/메소드를 사용할 수 없지만 내 자식 클래스는 사용할 수 있게 하고 싶을 경우 private 이 아닌 protected 를 사용하면 된다.

private 는 선언한 클래스 내부에서만 사용할 수 있다. 상속을 한다고 해도 사용할 수 없다.

protected 를 사용해서 내 private 변수/메소드를 오직 내 자식 클래스들만 쓸수 있게 만들 수 있다.
