Amazon Relational Database Service (RDS)

RDS를 사용하는 이유: AWS 클라우드 에서 관계형 데이터베이스를 설치, 운영, 확장할 수 있는 웹서비스이다. RDS를 EC2위에 설치하면 RDS는 EC2위에서 동작한다.

Virtual Private Cloud(VPC) 는 사용자의 AWS 계정 전용 가상 네트워크이다. VPC는 AWS 클라우드에서 다른 가상 네트워크와 논리적으로 분리되어있다. Amazon EC2인스턴스 같은 AWS 리소스를 VPC에서 실행할 수 있다.

RDS 데이터베이스를 생성할때 다음 항목들을 작성해줘야 한다.

  • DB 인스턴스 식별자 : 데이터베이스 인스턴스 이름
  • 마스터 사용자 이름 : 데이터베이스 username
  • 마스터 암호 : 데이터베이스 password

데이터 베이스 인스턴스를 생성하면 앤드포인트와 포트번호를 알 수 있다.

  • Endpoint: 아마존 RDS 데이터베이스 인스턴스와 연결 하기위해 필요한 데이터이다. 쉽게 말해 데이터베이스의 접근 경로이다.
  • 포트번호: 각 데이터베이스마다 디폴트 포트번호를 가지고 있다. 데이터베이스 가 listen 하고있는 포트 번호를 의미한다.

브로드캐스트 주소 계산법

Posted on 2020-01-26 Edited on 2020-01-27 In Network

브로드캐스팅 주소 계산법

알아야할 용어들:

  • 네트워크 주소: 하나의 네트워크를 통칭하기 위해 사용하는 주소.
  • 브로드캐스 주소: 특정 네트워크에 속하는 모든 호스트/클라이언트 들이 듣게되는 주소를 의미한다. 네트워크 주소와는 반대로 특정 네트워크의 맨 마지막 주소를 브로드캐스르 주소로 사용한다.

IP주소는 네트워크 주소와 호스트주소로 나누어진다. IP주소를 네트워크 주소와 호스트 주소로 구분해 주는것이 서브넷 마스크(Subnet mask)이다. 추가적으로 서브넷은 추가적인 서브넷 서브 네트워크가 필요할 경우 IP주소 중 호스트 주소를 서브넷과 호스트주소(<네트워크주소><서브넷><호스트주소>) 로 나누어준다.

서브넷 마스크라고 불리는 이유는 서브넷 마스크를 이용해서 IP주소의 네트워크 주소를 구분하기 위해서 이다.

서브넷 마스크는 32비트 숫자로 구성된 IP주소이며, IP주소를 네트워크 주소와 호스트 주소로 나눈다. 서브넷 마스크는 32비트 숫자들 중 네트워크 주소를 모두 1로 표현하고 호스트 주소를 0으로 표기한다. 주어진 네트워크에서 두개의 호스느 주소는 특정 목적을 위해 예약되어있어서 호스트한테 할당될 수 없다. 0 인 주소는 네트워크 주소에 할당되어 있고, 255는 broadcast address이기 때문에 호스트에게 할당될 수 없다.

– AND 연산 –

1 AND 1 = 1

1 AND 0 = 0

0 AND 1 = 0

0 AND 0 = 0

IP주소란?

네트워크 환경에서 컴퓨터간 통시하기 위해 각 컴퓨터에 부여되는 네트워크 상의 주소이다.

맥 zsh에 ifconifg 라고 입력 하면 내 컴퓨터에 부여된 네트워크상의 주소가 나온다.

  • IPv4주소는 내 컴퓨터의 주소이다.
  • 192.168.219.144 가 있으면 192.168.219 는 네트워크 주소이고 144는 호스트 주소이다(내 컴퓨터에 부여된 호스트 IP 이다. 내가 서울특별시 강남구 논현동 134-3 201호에 살면 서울특별시 강남구 논현동 134-3이 네트워크 주소이고, 201호가 호스트IP가 된다.)
  • IPv4의 마지막 자리가 무조건 호스트 주소는 아니다. IP주소의 클래스에 따라 달라진다.
  • 공통적인 공유되는 주소가 네트워크 주소, 내 자신을 식별해주는 마지막 자리가 호스트주소 이다.

IPv4 는 IP주소를 32자리로 표현한 것이다

192.168.24.133 이라는 주소가 있을때, 이 주소는 2진수 를 10진수로 표기한 것이다.

10진수를 2진수로 표기하게 되면 0~2^32, 약 43억개의 IP 주소를 사용할 수 있는것이다. 이전에는 충분할거라고 생각했지만 요즘은 휴대용기기를 포함해 여러개의 기기를 가질수 있기 때문에 IP주소가 부족해졌다. 따라서 현재는 IPv4에서 IPv6표현식으로 대체하는 중이다

서버 배포

  1. 보안그룹 생성 : 보안그룹 생성 > 보안그룹 이름/설명/인바운드 입력

  2. 인스턴스 생성 : Ubuntu 18.04 생성 > 보안 그룹 구성에서 기존의 보안그룹/새로운 보안 그룹 선택

  3. 인스턴스를 생성하면서 Secret 키를 생성할 수 있는데, 생성을 하면 ~/.ssh 디렉토리에 저장. AWS에서 인스터스를 생성 하면 키만 받고, 자물쇠는 우리가 인스턴스를 생성할 때 AWS 에서 자물쇠를 넣어준다.(다시말해 우리는 열쇠만 가지고 있는것이다.)

  4. EC2 서버로 접근하기: ssh -i ~/.ssh/[보안키이름].pem [유저]@[서버 주소]. 이렇게 하면 원래는 서버로 접속을 할 수 있지만 처음 설정 할때는 접근할 수 없다. EC2에 인바운드와 아웃바운드 설정을 해줘야 한다. 인바운드 설정에 유형이라는 항목이 있는데, 항목 내부에 다양한 선택지들이 있다. 그중 SSH를 설정해야 하는데, SSH를 설정하는 이유는 사용자가 SSH를 이용해서 인스턴스에 접근을 해야하기 때문이다.

    • 인바운드 : 외부에서 인스턴스로 들어오는 요청이다
    • 아웃바운드 : 인스턴스에서 외부로 나가는 트래픽이다. 0.0.0.0 은 어디든 갈 수 있다

  5. 인바운드를 설정하고 나서 다시 서버에 접근을 할려고하면 WARNING: UNPROTECTED PRIVATE KEY 라고 나온다. 그 이유는 우리가 사용하고 하는 secret키에 대한 보안이 너무 취약하기 때문이다. ~/.ssh 파일에서 ls-al을 해보면 -rw-|r--|r--@ 라고 나온다.

    • 맨앞 - 에 d가 있으면 디렉토리라는 뜻이다.
    • r 는 read 를 read를 의미한다
    • w 는 write 를 의미한다
    • x 는 executable을 의미한다
    • u 는 user(사용자)를 의미한다
    • g 는 그룹을 의미한다
    • o 는 기타 를 의미한다.
    • (파일타입|user|group|other) || 링크수 || 소유자 || 그룹 || 파일크기 || 파일수정시간 || 파일이름

  6. chmod와 숫자를 이용해서 해당 파일의 권한 변경하기: 숫자를 이용해서 해당파일의 user|group|other의 권한을 한번에 변경할 수 있다.

    • rwx 가 있는데 r 은 2^2, w 는 2^1, x 는 2^0을 의미한다. 다시말해 rwx = 2^2 + 2^1 + 2^0 = 7 을 의미한다.
    • r-x 가 있는데 r 은 2^2, x 는 2^0을 의미한다. 다시말해 rwx = 2^2 + 2^0 = 5 을 의미한다.
    • 쉽게 말해 r 는 4, w 는 2, x 는 1을 의미한다.
    • 권한을 부여할때 다음과 같이 부여한다. chmod 777 a.py. 이는 즉 a.py파일에 rwx|rwx|rwx를 부여한는 것이다.
    • chmod 124 b.py 는 b.py파일에 –x|-w-|r– 권한을 부여하는것이다

추상 클래스

추상 클래스는 미완성 클래스라고 생각하면 된다. 클래스가 미완성 이라는 것은 멤버의 개수에 관계된 것이 아니라, 단지 미완성 메서드를 포함하고 있는 클래스 인것이다.

미완성 클래스로 완성된 제품을 만들 수 없이 추상클래스로 인스턴스를 생성할 수 없다. 추상클래슨은 상속을 통해서 자손클래스에 의해서만 완성될 수 있다.( 장고에서 class Meta: abstract = True 를 생각하면 된다. )

추상 클래스 자체로는 클래스로서의 역할을 다 못하지만, 새로운 클래스르 작성하는데 있어서 바탕이 되는 부모클래스 이다. 자식 클래스들이 부모클래스 한테서 상속을 받는 이유는 부모의 기능을 그대로 사용하기 위해서이다. 하지만 만약에 부모 클래스로 인스턴스를 생성할 필요가 없다고 치면 그때 추상 클래스를 사용하면 된다.

추상 클래스는 abstract를 클래스 앞에 붙여서 사용한다. 추상 클래스는 추상 메서드를 포함 하고 있다는것을 제외하고는 일반클래스와 다른점이 없다. 추상 클래스 에도 생성자가 있으며, 멤버변수와 메서드를 모두 사용할 수 있다.

추상 메서드

메서드의 구현부가 비어있을 경우 추상 메서드 라고 한다. 메서드를 미완성으로 보류하는 이유는 메서드의 내용이 상속받는 클래스에 따라 달라질 수 있기 때문에 조상 클래스에 메소드의 구현부를 비워두고, 나중에 자식 클래스가 추상 메서드를 상속 받아서 채워가는 형식이다.(모든 인스턴스들이 동일한 메서드를 가져야 하는데 부모 클래스 입장에서 당장 메서드를 사용할 수 없을 경우 비워두는 것이다.)

추상 클래스로 부터 상속받는 자손 클래스는 오버라이딩을 통해 조상인 추상클래스의 추상메서드를 모두 구현해주어야 한다. 만일 조상으로부터 추상 메서드 중 하나라도 구현하지 않는다면, 자손 클래스 역시 추상 메서드로 지정해줘야한다(다음 자식 이 상속 받아서 지정하고 사용)
